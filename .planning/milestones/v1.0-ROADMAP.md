# Milestone v1.0: MVP — Dicticus macOS

**Status:** SHIPPED 2026-04-18
**Phases:** 1-5 (including 2.1 inserted)
**Total Plans:** 17

## Overview

Dicticus v1.0 delivers a fully local macOS dictation app: push-to-talk hotkeys, on-device ASR (Parakeet TDT v3 via FluidAudio on Apple Neural Engine), optional AI cleanup (Gemma 3 1B via llama.cpp), and DMG distribution. Five phases plus one inserted engine swap phase, executed over 4 days.

## Milestone Summary

**Decimal Phases:**
- Phase 2.1: ASR Engine Swap (inserted after Phase 2 — user preferred Parakeet v3 quality over Whisper)

**Key Decisions:**
- FluidAudio + Parakeet TDT v3 replaces WhisperKit + Whisper (better de/en quality, 5-10x faster, 8x less memory)
- llama.cpp for LLM inference (not MLX) — future Windows portability
- Unsandboxed DMG distribution — App Store sandbox blocks global hotkeys and text injection
- NSEvent global monitor replaces CGEventTap for modifier hotkeys — macOS 15 disables CGEventTap for ad-hoc signed apps
- Paste-at-cursor via NSPasteboard + CGEvent Cmd+V — most reliable cross-app method
- xcodegen for reproducible project generation from declarative project.yml

**Issues Resolved:**
- Data race on llama.cpp C pointers in CleanupService (isInferring guard)
- Prompt injection via ASR text in CleanupPrompt (control token sanitization)
- CGEventTap permanently disabled by macOS 15 (switched to NSEvent global monitor)
- Team ID mismatch on embedded frameworks in DMG (deep re-signing)

**Issues Deferred (Tech Debt):**
- APP-03 partial: transcribing/cleaning icon states not reactive (@State vs @StateObject)
- No duplicate modifier combo validation
- Downloaded GGUF model has no SHA256 integrity check
- Hardcoded HuggingFace URL for model download
- 50ms clipboard restore delay may be too short for Electron apps
- Mixed-language AI cleanup unreliable with Gemma 3 1B (accepted limitation)
