### Partial: APP-03

Recording indicator (red `mic.fill`) works correctly — this satisfies the core "visual recording indicator while push-to-talk is active" requirement. However, the extended icon state machine (transcribing → `waveform.circle`, cleaning → `sparkles`) is not reactive because `TranscriptionService` and `CleanupService` are held as `@State` in `DicticusApp.swift` instead of `@StateObject`. SwiftUI does not observe `@Published` changes on `@State`-held `ObservableObject` references.

**Impact:** Cosmetic. The user sees red mic during recording (correct), but the icon doesn't change during the brief transcribing/cleanup phase — it returns directly to idle `mic`. Core dictation and text injection are unaffected.

**Fix:** Change `@State private var transcriptionService` and `cleanupService` to use `@ObservedObject` or restructure to let `HotkeyManager` publish these states.
