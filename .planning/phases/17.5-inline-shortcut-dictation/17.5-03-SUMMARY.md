## Decisions Made
- Used legacy `openURL:` selector for responder-chain URL opening (the modern `openURL:options:completionHandler:` requires dictionary parameter that `perform(_:with:)` cannot pass from keyboard extensions)
- Smart text insertion lowercases first character mid-sentence but preserves acronyms detected by checking if second character is also uppercase
- Grace period of 0.5s matches KeyVox default for warm-start Darwin notification acknowledgment before falling back to URL launch
- Long-press gesture (0.5s) on mic button cancels active dictation to give users an escape hatch
