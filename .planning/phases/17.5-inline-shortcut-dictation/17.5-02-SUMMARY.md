## Files Created/Modified
- `iOS/Dicticus/IPC/DicticusIPCBridge.swift` - Shared IPC constants, UserDefaults keys, Darwin notification names, heartbeat helpers, state read/write (compiled into both targets)
- `iOS/Dicticus/IPC/DicticusHostBridge.swift` - Main app Darwin notification bridge with command callbacks, state publishers, and 1Hz heartbeat timer
- `iOS/Dicticus/DictationViewModel.swift` - Added hostBridge property and publish calls at recording start, transcribing, transcription ready, and no-speech/error paths
- `iOS/Dicticus/DicticusApp.swift` - Added hostBridge @StateObject, wired callbacks to viewModel, registered observers, added dicticus://record/start URL handler
- `iOS/project.yml` - Added DicticusIPCBridge.swift to DicticusKeyboard target sources

## Decisions Made
- DicticusHostBridge uses `@MainActor` with `nonisolated static let darwinCallback` and `Unmanaged.passUnretained` pointer pattern (matches KeyVox exactly)
- `publishTranscriptionReady` writes text to UserDefaults BEFORE posting the Darwin notification to prevent keyboard reading stale/empty data
- Heartbeat timer runs continuously so keyboard can always check `isSessionWarm()` even between dictation sessions
- DicticusIPCBridge.swift imports only Foundation (no UIKit/SwiftUI) to compile cleanly in both app and extension targets
- Old `kbSource`/`kbResult`/`kbResultReady` polling code left in place alongside new Darwin IPC calls for backward compatibility
