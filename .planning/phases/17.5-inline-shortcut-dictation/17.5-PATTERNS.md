**Pattern constraint (HIGH confidence):** `openAppWhenRun = true` must stay. AVAudioSession microphone recording fails with error 561015905 when the app is not foreground. The `@available(*, deprecated)` suppressor is the correct backward-compat pattern — keep it exactly as is.

## Shared Patterns

### App Group UserDefaults Flag Pattern
**Source:** `iOS/Dicticus/DictationViewModel.swift` (lines 94-95, 197-207)
**Apply to:** `DictateIntent.swift` (write), `DictationViewModel.swift` (read/clear), `DicticusApp.swift` (read-only guard)

The project-wide convention for cross-process signaling:
```swift
// Write (in intent or URL handler):
UserDefaults(suiteName: "group.com.dicticus")?.set(true, forKey: "someKey")

// Read + consume atomically (in view-model):
let shared = UserDefaults(suiteName: "group.com.dicticus")
if shared?.bool(forKey: "someKey") == true {
    shared?.set(false, forKey: "someKey")   // clear immediately on read
    // act on flag
}
```

The new `isShortcutLaunch` key follows this exact convention. Always clear the key immediately after reading it in `checkPendingIntent()`.

### NotificationCenter Observer Pattern
**Source:** `iOS/Dicticus/DictationViewModel.swift` (lines 171-195), `iOS/Dicticus/Intents/DictateIntent.swift` (line 19)
**Apply to:** No new observers needed in this phase — existing `.startDictation` and `.stopDictation` observers handle all triggers.

```swift
// Post (in intent):
NotificationCenter.default.post(name: .startDictation, object: nil)

// Observe (in view-model):
NotificationCenter.default.addObserver(
    forName: .startDictation,
    object: nil,
    queue: .main
) { [weak self] _ in
    Task { @MainActor in

}
```

### @Published State Flag Pattern

### Conditional UI Pattern (State-Driven)
**Source:** `iOS/Dicticus/DictationView.swift` (lines 103-134)
**Apply to:** `DictationView.swift` toolbar suppression and "Swipe back" hint

Shortcut-mode UI changes are expressed as conditional computed properties or inline `if viewModel.isShortcutLaunch` guards inside the body, not as separate Views. This matches how `modelMissing`, `warmupService.isWarming`, and `viewModel.state` currently drive the UI.

## Key Constraint Summary (for planner)

1. **`openAppWhenRun = true` must not be removed** from `DictateIntent`. This is a hard iOS AVAudioSession constraint, not a style choice.
2. **No programmatic app minimization exists** on iOS. The "auto-return" UX is: record → copy → show "Swipe up" hint → user swipes. Do not plan any `UIApplication.suspend` or URL-scheme-back approach.
3. **Live Activity must start before `startRecording()`** — the current order in `startDictation()` (Step 2 before Step 3) must be preserved.
4. **WhatsNew suppression must read `pendingDictation` before clearing it** — `checkPendingIntent()` clears the flag, so `DicticusApp.onAppear` must check it before `setupNotificationObserver()` fires.
5. **`isShortcutLaunch` must be reset to `false` in `stopDictation()`** — so manual dictations after a shortcut session do not inherit shortcut chrome.
