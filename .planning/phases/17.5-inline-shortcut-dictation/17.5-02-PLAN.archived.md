Add shortcut-mode UI behavior: hide settings chrome during shortcut-triggered dictation, show "swipe back" hint after completion, and prevent WhatsNew sheet from blocking auto-start.

Purpose: Minimizes friction when dictating via Action Button shortcut. The user sees only the recording UI (no navigation distractions) and gets clear guidance on how to return to their previous app after text is copied to clipboard.

Output: Modified DictationView with conditional shortcut-mode UI, modified DicticusApp with WhatsNew suppression guard. Verified on device via human checkpoint.

  <action>
    **DictationView.swift** -- Three changes:

    1. **Hide toolbar in shortcut mode.** Replace the toolbar block (lines 84-90):
    ```swift
    .toolbar {
        if !viewModel.isShortcutLaunch {
            ToolbarItem(placement: .topBarTrailing) {
                Button(action: { showingSettings = true }) {
                    Image(systemName: "gear")
                }
            }
        }
    }
    ```

    2. **Add "Swipe up to return" hint after result in shortcut mode.** Inside the `if let result = viewModel.lastResult` VStack (after the Text(result) block at line 63, before the closing `}`), add:
    ```swift
    if viewModel.isShortcutLaunch {
        Label("Swipe up to return to your app", systemImage: "arrow.up")
            .font(.caption)
            .foregroundStyle(.secondary)
            .padding(.top, 4)
    }
    ```

    3. **Update statusLabel computed property.** In the `.idle` case of statusLabel (line 124), make it context-aware:
    ```swift
    case .idle:
        if viewModel.isShortcutLaunch && viewModel.lastResult != nil {
            return "Copied to clipboard"
        }
        return "Ready"
    ```
    This shows "Copied to clipboard" as the status after a shortcut dictation completes, instead of generic "Ready".

    **DicticusApp.swift** -- One change:

    **Suppress WhatsNew sheet when shortcut dictation is pending.** Replace the `.onAppear` block (lines 42-46):
    ```swift
    .onAppear {
        let pendingDictation = UserDefaults(suiteName: "group.com.dicticus")?.bool(forKey: "pendingDictation") ?? false
        if !hasSeenWhatsNewV2 && !pendingDictation {
            showingWhatsNew = true
        }
    }
    ```
    This prevents the WhatsNew sheet from appearing when the app is launched via the shortcut intent. The sheet will still show on the next manual app launch when `hasSeenWhatsNewV2` is still false.
  </action>
