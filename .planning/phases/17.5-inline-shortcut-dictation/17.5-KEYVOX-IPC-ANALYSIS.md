# KeyVox IPC Architecture Analysis

Source: https://github.com/macmixing/keyvox/ (commit on main as of 2026-04-23)

## 1. Architecture Overview

KeyVox uses a **3-layer IPC architecture** for keyboard-to-app communication:

1. **KeyVoxIPCBridge** (shared enum) -- defines all constants, read/write helpers
2. **KeyboardIPCManager** (keyboard extension) -- sends commands TO app, receives state FROM app
3. **KeyVoxKeyboardBridge** (main app) -- receives commands FROM keyboard, publishes state back

Communication uses **Darwin notifications** (CFNotificationCenterGetDarwinNotifyCenter) for signaling and **App Group UserDefaults** for data transfer. Audio meter data uses a **shared file** for high-frequency updates.

Key insight: Darwin notifications carry NO payload. They are pure signals. All data is exchanged via UserDefaults or files in the shared container.

### Heartbeat Mechanism

```swift
static let heartbeatFreshnessWindow: TimeInterval = 5  // session is "warm" if heartbeat < 5s old

static func isSessionWarm() -> Bool {
    guard let d = defaults else { return false }
    let ts = d.double(forKey: Key.sessionTimestamp)
    guard ts > 0 else { return false }
    return Date().timeIntervalSince1970 - ts < heartbeatFreshnessWindow
}

// Called at 1Hz by the app's heartbeat timer, rate-limited to avoid thrashing
private static var lastHeartbeatUpdateTime: TimeInterval = 0
static func touchHeartbeat() {
    let now = Date().timeIntervalSince1970
    guard now - lastHeartbeatUpdateTime >= 1.0 else { return }
    lastHeartbeatUpdateTime = now
    defaults?.set(now, forKey: Key.sessionTimestamp)
}
```

### Live Audio Meter (Binary File IPC)

For high-frequency audio level data, KeyVox writes a 20-byte binary file to the shared container instead of using UserDefaults:

```swift
// File: "live-meter-state.bin" in App Group container
// Format: [version:UInt8][signalState:UInt8][pad:UInt16][level:Float32][sequence:UInt32][timestamp:Float64]
// Written atomically, read by keyboard for audio indicator animation
```

Not essential for Dicticus MVP but useful for visual audio feedback in the keyboard.

## 3. Keyboard IPC Manager (Extension Side)

File: `iOS/KeyVox Keyboard/Core/Transport/KeyboardIPCManager.swift`

### Callback-Based API

```swift
final class KeyboardIPCManager {
    // State callbacks -- set by KeyboardDictationController
    var onRecordingStarted: (() -> Void)?
    var onTranscribingStarted: (() -> Void)?
    var onTranscriptionReady: ((String) -> Void)?
    var onNoSpeech: (() -> Void)?
}
```

### Commands Sent TO App

```swift
func sendStartCommand() {
    setRecordingState("waitingForApp")           // Write state to UserDefaults
    postDarwinNotification(named: .startRecording) // Signal app
}

func sendStopCommand() {
    setRecordingState("transcribing")
    postDarwinNotification(named: .stopRecording)
}

func sendCancelCommand() {
    KeyVoxIPCBridge.clearTransientOperationState()
    KeyVoxIPCBridge.setRecordingState("idle")
    postDarwinNotification(named: .cancelRecording)
}
```

### Receiving Transcription Text

When `transcriptionReady` notification fires, the manager reads text from UserDefaults:

```swift
case KeyVoxIPCBridge.Notification.transcriptionReady:
    guard let text = self.latestTranscription(), !text.isEmpty else {
        self.onNoSpeech?()  // Treat empty transcription as no-speech
        return
    }
    self.onTranscriptionReady?(text)
```

### Stale State Reconciliation

Critical for reliability -- detects when the app crashed mid-session:

```swift
func reconcileStaleSharedStateIfNeeded() -> SharedRecordingState {
    let state = currentSharedRecordingState()
    switch state {
    case .idle:
        return .idle
    case .waitingForApp:
        // Stale after 5 seconds
        if let age = KeyVoxIPCBridge.currentRecordingStateAge(), age > 5 {
            KeyVoxIPCBridge.clearTransientOperationState()
            return .idle
        }
        return .waitingForApp
    case .recording, .transcribing:
        // Only valid if heartbeat is fresh
        guard !isSessionWarm() else { return state }
        KeyVoxIPCBridge.clearTransientOperationState()
        return .idle
    }
}
```

### Darwin Notification Registration Pattern

```swift
private func registerDarwinObserver(named name: String) {
    let center = CFNotificationCenterGetDarwinNotifyCenter()
    CFNotificationCenterAddObserver(
        center,
        Unmanaged.passUnretained(self).toOpaque(),  // observer pointer
        Self.notificationCallback,                    // C function callback
        name as CFString,
        nil,
        .deliverImmediately
    )
}

// Static C callback -- bridges to instance method
nonisolated private static let notificationCallback: CFNotificationCallback = {
    _, observer, name, _, _ in
    guard let observer, let rawName = name?.rawValue as String? else { return }
    let manager = Unmanaged<KeyboardIPCManager>.fromOpaque(observer).takeUnretainedValue()
    manager.handleNotification(named: rawName)
}

// Instance method dispatches to main queue
private func handleNotification(named name: String) {
    DispatchQueue.main.async { [weak self] in
        guard let self else { return }
        switch name {
        case KeyVoxIPCBridge.Notification.transcriptionReady:
            // ... read from UserDefaults
        }
    }
}
```

## 4. Host App Bridge (Main App Side)

File: `iOS/KeyVox iOS/App/Integration/KeyVoxKeyboardBridge.swift`

### Lifecycle

```swift
func registerObservers() {
    // Listen for keyboard commands
    registerDarwinObserver(named: .startRecording)
    registerDarwinObserver(named: .stopRecording)
    registerDarwinObserver(named: .cancelRecording)
    registerDarwinObserver(named: .disableSession)
    
    touchHeartbeat()          // Write initial heartbeat
    startHeartbeatTimer()     // Start 1Hz heartbeat
}
```

### Heartbeat Timer

```swift
private func startHeartbeatTimer() {
    heartbeatTimer = Timer.scheduledTimer(withTimeInterval: 1, repeats: true) { [weak self] _ in
        self?.touchHeartbeat()
    }
}
```

### Publishing State Updates TO Keyboard

```swift
func publishRecordingStarted() {
    KeyVoxIPCBridge.setRecordingState("recording")
    KeyVoxIPCBridge.writeLiveMeter(level: 0, signalState: .dead)
    postDarwinNotification(named: .recordingStarted)
    KeyVoxIPCBridge.touchHeartbeat()
}

func publishTranscribing() {
    KeyVoxIPCBridge.setRecordingState("transcribing")
    postDarwinNotification(named: .transcribingStarted)
}

func publishTranscriptionReady(_ text: String) {
    KeyVoxIPCBridge.setTranscription(text)         // Write text to UserDefaults FIRST
    KeyVoxIPCBridge.setRecordingState("idle")
    KeyVoxIPCBridge.clearLiveMeter()
    postDarwinNotification(named: .transcriptionReady) // THEN signal keyboard
}

func publishNoSpeech() {
    KeyVoxIPCBridge.clearTransientOperationState()
    KeyVoxIPCBridge.setRecordingState("idle")
    postDarwinNotification(named: .noSpeech)
}
```

### Command Callbacks

```swift
var onStartRecordingCommand: (() -> Void)?
var onStopRecordingCommand: (() -> Void)?
var onCancelRecordingCommand: (() -> Void)?
var onDisableSessionCommand: (() -> Void)?
```

These are wired to `TranscriptionManager` in the main app.

## 5. Keyboard Dictation Controller (State Machine)

File: `iOS/KeyVox Keyboard/Core/Dictation/KeyboardDictationController.swift`

### States

```swift
enum KeyboardState: Equatable {
    case idle
    case waitingForApp
    case recording
    case transcribing
    case preparingPlayback  // TTS only
    case speaking           // TTS only
    case pausedSpeaking     // TTS only
}
```

For Dicticus, only idle/waitingForApp/recording/transcribing are relevant.

### State Transitions on Mic Tap

```swift
func handleMicTap() {
    syncStateFromSharedState()  // Reconcile stale state first

    switch state {
    case .idle:
        state = .waitingForApp
        scheduleWaitingTimeout()

        if ipcManager.isSessionWarm() {
            // App is alive -- send command directly via Darwin notification
            ipcManager.sendStartCommand()
            scheduleWarmSessionGracePeriod(after: effectiveWarmSessionGracePeriod())
        } else {
            // App not running -- launch it via URL scheme
            openContainingApp(startRecordingURL)
        }

    case .recording:
        state = .transcribing
        ipcManager.sendStopCommand()

    case .waitingForApp, .preparingPlayback, .transcribing:
        break  // Ignore taps during transitions
    }
}
```

### Cold Start vs Warm Start

**Cold start (app not running):**
1. Keyboard opens URL `keyvoxios://record/start`
2. iOS launches the main app
3. App's URL handler calls `handleStartRecordingCommand(isFromURL: true)`
4. App starts recording, calls `keyboardBridge.publishRecordingStarted()`
5. Darwin notification fires, keyboard receives `onRecordingStarted`
6. Keyboard transitions from `waitingForApp` to `recording`

**Warm start (app already running, heartbeat fresh):**
1. Keyboard sends `sendStartCommand()` -- writes "waitingForApp" + posts Darwin notification
2. App receives `onStartRecordingCommand`, starts recording
3. App publishes `recordingStarted` via Darwin notification
4. Keyboard receives it, transitions to `recording`
5. Grace period: if app doesn't respond within 0.5s, falls back to URL launch

### Warm Session Grace Period

```swift
private func scheduleWarmSessionGracePeriod(after delay: TimeInterval) {
    gracePeriodAction = scheduleAction(delay) { [weak self] in
        guard let self, self.state == .waitingForApp else { return }
        // App didn't respond in time -- fall back to URL launch
        guard self.ipcManager.currentRecordingState() != .recording else { return }
        self.openContainingApp(self.startRecordingURL)
    }
}

// Grace periods:
// Normal: 0.5s
// After TTS playback: 0.5s
// With Bluetooth audio: 1.5s (BT routing takes longer)
```

### Waiting Timeout

```swift
// If app doesn't start recording within 5 seconds, reset to idle
private func scheduleWaitingTimeout() {
    waitingForAppTimeoutAction = scheduleAction(waitingTimeoutDuration) { [weak self] in
        guard let self, self.state == .waitingForApp else { return }
        self.state = .idle
    }
}
```

### Cancel Flow

```swift
func handleCancelTap() {
    cancelWaitingTimeout()
    ipcManager.sendCancelCommand()
    state = .idle
}
```

## 6. Text Insertion into textDocumentProxy

File: `iOS/KeyVox Keyboard/Core/Input/KeyboardTextInputController.swift`

The transcription text goes through two heuristic transforms before insertion:

### Step 1: Capitalization Normalization

```swift
// If the transcription starts with a capital letter (ASR default),
// and the cursor is mid-sentence, lowercase the first letter.
// Preserves capitalization at sentence starts and for proper nouns.
let capitalizationNormalizedText = KeyboardInsertionCapitalizationHeuristics
    .normalizeLeadingCapitalizationIfNeeded(
        text: cleanedText,
        documentContextBeforeInput: documentProxy.documentContextBeforeInput,
        shouldPreserveLeadingCapitalization: shouldPreserveLeadingCapitalization
    )
```

Logic: If the first word is `[A-Z][a-z]+` (default sentence case, not an acronym or proper noun), and the context before cursor does NOT end with `.?!\n` or is empty, lowercase it.

### Step 2: Smart Leading Space

```swift
// Adds a space before the transcription if needed
let insertionText = KeyboardInsertionSpacingHeuristics.applySmartLeadingSeparatorIfNeeded(
    to: capitalizationNormalizedText,
    documentContextBeforeInput: documentProxy.documentContextBeforeInput
)
```

Logic: Adds a leading space if the previous character is alphanumeric or closing punctuation, and the first character of the transcription is not whitespace or closing punctuation.

### Final Insertion

```swift
documentProxy.insertText(insertionText)
```

### Trailing newline cleanup

```swift
let cleanedText = text.replacingOccurrences(
    of: #"[\r\n]+$"#, with: "", options: .regularExpression
)
```

## 7. Containing App Launcher (URL Scheme from Extension)

File: `iOS/KeyVox Keyboard/App/KeyboardContainingAppLauncher.swift`

Keyboard extensions cannot use `UIApplication.shared.open()`. KeyVox uses the responder chain trick:

```swift
func open(_ url: URL?) {
    guard let url else { return }
    let modernSelector = NSSelectorFromString("openURL:options:completionHandler:")
    let legacySelector = NSSelectorFromString("openURL:")

    var responder = responderProvider()  // starts from self (UIInputViewController)
    while let currentResponder = responder {
        if currentResponder.responds(to: modernSelector) {
            _ = currentResponder.perform(modernSelector, with: url, with: nil)
            return
        }
        if currentResponder.responds(to: legacySelector) {
            _ = currentResponder.perform(legacySelector, with: url)
            return
        }
        responder = currentResponder.next
    }
}
```

URL scheme: `keyvoxios://record/start`

## 8. Main App -- TranscriptionManager Integration

File: `iOS/KeyVox iOS/Core/Transcription/TranscriptionManager.swift`

The `TranscriptionManager` owns `keyboardBridge: KeyVoxKeyboardBridge` and calls its publish methods at each lifecycle point:

```swift
// On successful recording start:
keyboardBridge.publishRecordingStarted()

// On transcription start:
keyboardBridge.publishTranscribing()

// On transcription complete:
keyboardBridge.publishTranscriptionReady(finalText)  // or publishNoSpeech()
```

Session management: The app keeps the audio session alive (monitoring mode) between dictations. The keyboard can check `isSessionWarm()` to see if the app is ready for instant start.

### Session Policy

```swift
struct SessionPolicy {
    var idleTimeout: TimeInterval?             // 300s (5 min) -- auto-shutdown after idle
    var noSpeechAbandonmentTimeout: TimeInterval? // 45s
    var postSpeechInactivityTimeout: TimeInterval? // 180s
    var emergencyUtteranceCap: TimeInterval?       // 900s (15 min)
}
```

## 9. Complete Flow Diagram

```
COLD START:
Keyboard mic tap
  -> state = waitingForApp
  -> openContainingApp("keyvoxios://record/start")
  -> [iOS launches app]
  -> App: handleStartRecordingCommand(isFromURL: true)
  -> App: recorder.startRecording()
  -> App: keyboardBridge.publishRecordingStarted()
      -> UserDefaults: recordingState = "recording"
      -> Darwin: "recordingStarted"
  -> Keyboard: onRecordingStarted -> state = recording
  -> [user speaks]
  -> Keyboard mic tap
  -> state = transcribing
  -> ipcManager.sendStopCommand()
      -> UserDefaults: recordingState = "transcribing"
      -> Darwin: "stopRecording"
  -> App: onStopRecordingCommand
  -> App: recorder.stopRecording()
  -> App: transcriptionService.run(audioFrames)
  -> App: keyboardBridge.publishTranscribing()
      -> Darwin: "transcribingStarted"
  -> App: keyboardBridge.publishTranscriptionReady(text)
      -> UserDefaults: latestTranscription = text
      -> UserDefaults: recordingState = "idle"
      -> Darwin: "transcriptionReady"
  -> Keyboard: onTranscriptionReady(text)
  -> textInputController.insertTranscription(text)
      -> smart capitalization + spacing
      -> textDocumentProxy.insertText(processedText)
  -> state = idle

WARM START:
Keyboard mic tap
  -> isSessionWarm() == true
  -> ipcManager.sendStartCommand()
      -> UserDefaults: recordingState = "waitingForApp"
      -> Darwin: "startRecording"
  -> scheduleWarmSessionGracePeriod(0.5s)
  -> App receives Darwin notification immediately
  -> [same flow as above from publishRecordingStarted()]
  -> [grace period cancelled when recordingStarted arrives]
```

### What Dicticus Already Has vs What's Needed

**Already exists:**
- App Group "group.com.dicticus" configured
- UserDefaults keys: "kbResult", "kbResultReady" (polling-based -- replace with Darwin notifications)
- `DictationViewModel` with working ASR pipeline
- Keyboard extension with `textDocumentProxy` access
- URL scheme likely needs registration

**Must build:**
- `DicticusIPCBridge.swift` -- shared constants + helpers (add to both targets)
- `DicticusKeyboardIPCManager.swift` -- keyboard-side Darwin notification observer/sender
- `DicticusHostBridge.swift` -- app-side Darwin notification observer/sender + heartbeat timer
- `DicticusKeyboardDictationController.swift` -- state machine in keyboard
- Wire `DicticusHostBridge` into `DictationViewModel`
- Register `dicticus://` URL scheme in Info.plist
- URL handler in AppDelegate/SceneDelegate
- Smart text insertion (capitalization + spacing heuristics)
- Responder-chain URL opener in keyboard extension

### What to Skip for MVP

- Live audio meter binary file IPC (nice-to-have visual feedback)
- TTS integration
- Bluetooth audio route detection
- Session disable timing
- Interrupted capture recovery
- Dictionary hint prompts
- Onboarding state reporting
