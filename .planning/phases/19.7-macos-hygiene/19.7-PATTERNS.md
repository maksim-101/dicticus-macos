**Analog:** `scripts/build-dmg.sh`

**What changes:** New file. Borrow shell idioms (shebang, `set -euo pipefail`, `=== Step N: ===` headers, error gating, friendly final summary). Does NOT need to re-sign — it just relocates an already-signed `.app`.

    -configuration Release \

**Existing @Published + status enum pattern** (`PermissionManager.swift:50-66`) — mirror exactly for the new property:
```swift
@MainActor
class PermissionManager: ObservableObject {
    @Published var microphoneStatus: PermissionStatus = .pending
    @Published var accessibilityStatus: PermissionStatus = .pending
    @Published var hasCompletedOnboarding = false

    private static let onboardingKey = "hasCompletedOnboarding"
    private var pollTimer: Timer?

    /// True when both required permissions are .granted.
    /// Input Monitoring is NOT needed — KeyboardShortcuts uses Carbon RegisterEventHotKey,
    /// not NSEvent.addGlobalMonitorForEventsMatchingMask.
    var allGranted: Bool {
        microphoneStatus == .granted &&
        accessibilityStatus == .granted

```

**Existing OS-status → PermissionStatus mapping** (`PermissionManager.swift:69-84`) — copy the switch-shape for the IOHID check:
```swift
func checkAll() {
    let audioStatus = AVCaptureDevice.authorizationStatus(for: .audio)
    switch audioStatus {
    case .authorized:           microphoneStatus = .granted
    case .denied, .restricted:  microphoneStatus = .denied
    case .notDetermined:        microphoneStatus = .pending
    @unknown default:           microphoneStatus = .pending
    }

    let axTrusted = AXIsProcessTrusted()
    accessibilityStatus = axTrusted ? .granted : .pending
}
```
For Input Monitoring, `IOHIDCheckAccess` returns `kIOHIDAccessTypeGranted` / `kIOHIDAccessTypeDenied` / `kIOHIDAccessTypeUnknown` — map to `.granted` / `.denied` / `.pending` with the same switch shape.

**Existing request-then-rely-on-polling pattern** (`PermissionManager.swift:95-100`) — `requestInputMonitoring()` should call `IOHIDRequestAccess(kIOHIDRequestTypeListenEvent)` and NOT touch `@Published` state directly; the 2s timer picks up the change:
```swift
func requestAccessibility() {

    AXIsProcessTrustedWithOptions(options as CFDictionary)
    // Do not update status here — polling picks it up within 2 seconds
}
```

**Polling loop is reused as-is** (`PermissionManager.swift:104-113`) — no changes; the new property is checked inside the existing `checkAll()` invocation.

**Module-level captured-key pattern for C globals** (`PermissionManager.swift:42-43`) — relevant if `kIOHIDRequestTypeListenEvent` triggers Swift 6 concurrency warnings:

## Shared Patterns

### Shell script idioms (applies to all 3 new scripts)
**Source:** `scripts/build-dmg.sh:1-26`, `scripts/verify-memory.sh:1-15`
**Apply to:** `install-local.sh`, `uninstall.sh`, `generate-icons.sh`
```bash
#!/bin/bash
# <One-line purpose.>
#
# Usage:
#   ./scripts/<name>.sh
#
# <Optional: prerequisites or output description.>
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
PROJECT_DIR="$(dirname "$SCRIPT_DIR")/macOS"   # if needed
OUTPUT_DIR="$(dirname "$SCRIPT_DIR")"
```
Every step echoes `=== Step N: <Title> ===`. Every destructive operation is preceded by an existence/precondition guard with a clear `ERROR:` message and `exit 1`. Final block is a friendly summary of what was done.

### @MainActor service with @Published state + 2s polling
**Source:** `PermissionManager.swift:50-119`
**Apply to:** any service exposing OS-level liveness state (used here for the new `inputMonitoringStatus`, optionally for a new `MultiInstallDetector` if planner chooses that split).
- `@MainActor class` conforming to `ObservableObject`
- `@Published` properties initialized to a sensible non-fatal default (`.pending`)
- A single `checkAll()` (or `checkOnce()`) entry point that does the OS calls synchronously
- A `startPolling()` method gated by `guard pollTimer == nil else { return }`
- Timer closure uses `[weak self]` and dispatches via `Task { @MainActor in self?.checkAll() }`
- Mutation happens only inside `@MainActor`-isolated code; never publish from a callback thread

### Error-logged-but-non-fatal pattern
**Source:** `ModifierHotkeyListener.swift:94, 122-127`
**Apply to:** new KeyboardShortcuts registration-liveness capture in `HotkeyManager` (D-04 layer 2)
```swift
let log = Logger(subsystem: "com.dicticus", category: "<feature>")
// ... attempt to set up an OS resource ...
if resource != nil {
    log.info("<resource> started for <event>")
} else {
    log.error("<resource> creation FAILED — <suspected cause>")
}
```
Always `Logger(subsystem: "com.dicticus", category: "<feature-kebab>")` — not `print`, not `os_log`. Matches the project's existing observability surface.

### Settings deep-link
**Source:** `SystemSettingsURLs.swift:7-19` and `PermissionRow.swift:39-49`
**Apply to:** Repair banner (D-05) and `.denied` action button on every permission row (D-09)
```swift
Button(status == .denied ? "Open Settings" : "Grant Access") {
    if status == .denied {
        SystemSettingsURL.open(settingsURL)
    } else {
        grantAction()
    }
}
.controlSize(.small)

```
Per D-09, change the `.denied` label from `"Open Settings"` to **`"Repair in System Settings"`** when the row is in `.denied` state — the rest of the pattern (controlSize, fixedSize, branch on `status`) is preserved.
