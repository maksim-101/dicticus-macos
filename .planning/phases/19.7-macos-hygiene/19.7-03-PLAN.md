    /// Run `mdfind kMDItemCFBundleIdentifier == 'com.dicticus.macos'` once and publish the result.
    /// Designed to be invoked from MenuBarView.onAppear, NOT from the 2s polling timer (D-07).
    /// Excludes nothing — the view layer decides which copies are "stale" vs "canonical".
    func checkMultipleInstalls() {
        let task = Process()
        task.executableURL = URL(fileURLWithPath: "/usr/bin/mdfind")
        task.arguments = ["kMDItemCFBundleIdentifier == 'com.dicticus.macos'"]

        let pipe = Pipe()
        task.standardOutput = pipe
        task.standardError = Pipe()

        do {
            try task.run()
            task.waitUntilExit()
            let data = pipe.fileHandleForReading.readDataToEndOfFile()
            guard let output = String(data: data, encoding: .utf8) else {
                multipleDicticusCopies = []
                return
            }
            let paths = output
                .split(separator: "\n")
                .map(String.init)
                .filter { !$0.isEmpty }
                .filter { $0.hasSuffix("/Dicticus.app") }
                .map { URL(fileURLWithPath: $0) }
            multipleDicticusCopies = paths
        } catch {
            multipleDicticusCopies = []
        }
    }

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-19.7-08 | Spoofing | Multi-copy paths display | accept | Paths come from mdfind on the local filesystem; not network-sourced. Display uses `.lineLimit(1) + .truncationMode(.middle)` so a maliciously crafted long path cannot push UI off-screen. |
| T-19.7-09 | Tampering | reregisterAll while a hotkey is firing | mitigate | Cancelling Task handles is safe; `handleKeyDown`/`handleKeyUp` operate on weak service refs; the existing `isKeyDown` guard prevents key-repeat replay. |
| T-19.7-10 | Information disclosure | os.Logger emissions | accept | Log lines contain no user data — only state transitions and stream lifecycle. Subsystem `com.dicticus` is consistent with the rest of the project. |
