# Phase 19.7 Plan 03: Hotkey Re-Authorization Recovery UX Summary

Sticky Repair banner + always-visible "Re-register hotkeys" button + launch-only multi-copy warning row close UAT row M1 — daily macOS dictation is unblocked when AX is missing post-onboarding, when KeyboardShortcuts AsyncStreams silently die, or when stale `.app` copies on disk poison TCC.

## What Changed

### Task 1 — HotkeyManager.swift (commit `d894091`)
- Added `import os` and module-level `Logger(subsystem: "com.dicticus", category: "hotkey-manager")`.
- Added `@Published var registrationFailed: Bool = false` next to existing published state.
- Added `private var plainDictationTask: Task<Void, Never>?` and `private var cleanupTask: Task<Void, Never>?` so AsyncStream consumers can be cancelled.
- Replaced the two untracked `Task { ... }` spawns inside `setup(...)` with handle-tracked + log-instrumented versions. The for-await loop body is unchanged; the shape now logs lifecycle on entry, logs an error if the stream ends, and flips `registrationFailed = true` on stream exit.
- Added `func reregisterAll()` after `setupModifierListener(...)`. Cancels both Task handles, restarts the modifier listener (`stop()` + `start()`), resets `registrationFailed`, then re-spawns both AsyncStream Tasks. No state-machine touchpoints (`handleKeyDown`, `handleKeyUp`, `bindState`, `cleanupService` setter) modified.

### Task 2 — PermissionManager.swift (commit `dd50d5e`)
- Added `@Published var multipleDicticusCopies: [URL] = []` after `hasCompletedOnboarding`.
- Added `func checkMultipleInstalls()` after `loadOnboardingState()`. Spawns `/usr/bin/mdfind` with the verbatim predicate `kMDItemCFBundleIdentifier == 'com.dicticus.macos'`, filters output to `/Dicticus.app`-suffixed paths, publishes the URL list. Empties the list on any decode/launch error.
- The `pollTimer` block is unchanged — no `checkMultipleInstalls()` call inside it (D-07 launch-only).

### Task 3 — HotkeySettingsView.swift (commit `0e55d70`)
- Injected `@EnvironmentObject var hotkeyManager: HotkeyManager` (provided by MenuBarView's environment).
- Appended a third HStack inside the existing `VStack(spacing: 4)` with the row label "Re-register hotkeys" and a small `.controlSize(.small) .fixedSize(horizontal: true, vertical: false)` button labelled "Re-register" that calls `hotkeyManager.reregisterAll()`. Always visible (no conditional).

### Task 4 — MenuBarView.swift (commit `f99a1ee`)
- Added `@State private var showMultiCopyDetail = false` next to `showOnboarding`.
- Inserted two conditional blocks at the very top of the dropdown content (inside the `else` branch of `if showOnboarding`, BEFORE `WarmupRow()`):
  1. **Repair banner** — gated by `(permissionManager.accessibilityStatus == .denied && permissionManager.hasCompletedOnboarding) || hotkeyManager.registrationFailed`. Red triangle icon, body title, "Repair" button → `SystemSettingsURL.open(SystemSettingsURL.accessibility)`, caption "Relaunch Dicticus after granting." indented under the icon. `Color.red.opacity(0.08)` background + `cornerRadius(6)`. Followed by `Divider()`.
  2. **Multi-copy warning** — gated by `permissionManager.multipleDicticusCopies.count > 1`. Orange triangle icon, body title, Show/Hide toggle button. When expanded, lists each path in caption text with `.lineLimit(1) + .truncationMode(.middle)` plus the footer "Run scripts/install-local.sh to consolidate." `Color.orange.opacity(0.08)` background. Followed by `Divider()`.
- Appended `permissionManager.checkMultipleInstalls()` to the existing `.onAppear` block (after `loadOnboardingState()`, before the `showOnboarding` decision).
- All other sections (`WarmupRow`, the existing 3-row PermissionRow VStack, `AiCleanupInfoView`, `HotkeySettingsView`, `LastTranscriptionView`, History button, `SettingsSection`, "Check for Updates", Quit) untouched.
