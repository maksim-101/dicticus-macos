# Phase 2.1: ASR Engine Swap (WhisperKit to FluidAudio + Parakeet TDT v3) - Research

**Researched:** 2026-04-16
**Domain:** Swift ASR pipeline migration -- FluidAudio SDK, CoreML, Parakeet TDT v3
**Confidence:** HIGH

## Summary

This phase replaces WhisperKit + Whisper large-v3-turbo with FluidAudio + Parakeet TDT v3 as the sole ASR engine. FluidAudio is a Swift SDK (Apache 2.0) distributed as a Swift Package (SPM), wrapping Parakeet CoreML models that run on Apple Neural Engine. The SDK provides an actor-based `AsrManager` that accepts `[Float]` audio samples at 16kHz mono and returns an `ASRResult` with `.text` and `.confidence`. FluidAudio does NOT provide its own audio recording abstraction -- the app must manage `AVAudioEngine` directly and feed samples to the SDK. FluidAudio DOES provide a Silero-based `VadManager` for voice activity detection that replaces WhisperKit's internal EnergyVAD.

A critical finding: **Parakeet TDT v3 does NOT output detected language codes.** The model auto-detects language internally for transcription quality, but `ASRResult` has no `language` property. Language detection must be implemented post-hoc using Apple's `NLLanguageRecognizer` from the NaturalLanguage framework on the transcribed text. This is a design change from the WhisperKit approach (which returned `result.language`).

**Primary recommendation:** Use `AsrManager` for batch transcription (not `SlidingWindowAsrManager`), manage audio recording via `AVAudioEngine` with format conversion to 16kHz mono Float32, use `NLLanguageRecognizer` for post-hoc language detection, and use FluidAudio's `VadManager` as the engine-level VAD layer.

## Architecture Patterns

### Service Architecture (Preserved from Phase 2)

### Pattern 1: AsrManager Lifecycle
**What:** Actor-based ASR manager with download-load-transcribe-cleanup lifecycle
**When to use:** All transcription operations
**Example:**
```swift
// Source: FluidAudio GitHub README + AsrTypes.swift source code
import FluidAudio

// 1. Download and load models (first run downloads ~2.69 GB from HuggingFace)
let models = try await AsrModels.downloadAndLoad(version: .v3)

// 2. Create manager with default config (16kHz, TDT decoder)
let asrManager = AsrManager(config: .default)

// 3. Load models into manager (triggers CoreML compilation on first run)
try await asrManager.loadModels(models)

// 4. Transcribe batch of 16kHz mono Float32 samples
let result = try await asrManager.transcribe(samples)
// result.text -> transcribed text
// result.confidence -> Float confidence score
// result.duration -> audio duration
// result.processingTime -> inference time
// result.rtfx -> real-time factor

// 5. Cleanup when done
await asrManager.cleanup()
```

### Pattern 3: Post-Hoc Language Detection via NLLanguageRecognizer
**What:** Detect language of transcribed text since Parakeet returns no language code
**When to use:** After successful transcription, before building DicticusTranscriptionResult
**Example:**
```swift
// Source: Apple NaturalLanguage framework docs
import NaturalLanguage

func detectLanguage(_ text: String) -> String {
    let recognizer = NLLanguageRecognizer()
    // Constrain to {de, en} -- Parakeet supports 25 languages but we only care about 2
    recognizer.languageConstraints = [.german, .english]
    recognizer.processString(text)
    
    if let language = recognizer.dominantLanguage {
        switch language {
        case .german: return "de"
        case .english: return "en"
        default: return "en"  // Fallback for unexpected values
        }
    }
    return "en"  // Default to English if detection fails
}
```

### Pattern 4: VadManager for Engine-Level VAD
**What:** Replace WhisperKit's EnergyVAD with Silero VAD v6 CoreML
**When to use:** Energy pre-filter layer (layer 2 of three-layer VAD defense)
**Example:**
```swift
// Source: FluidAudio VAD/GettingStarted.md
import FluidAudio

// Initialize VAD manager (downloads ~1.8 MB Silero model on first run)
let vadManager = try await VadManager(config: VadConfig(defaultThreshold: 0.75))

// Process recorded samples to check for voice activity
let vadResults = try await vadManager.process(audioSamples)

// Check if any chunk has speech probability above threshold
let hasVoice = vadResults.contains { $0.probability > 0.5 }

// Alternative: Use segmentation for speech boundary detection
var segConfig = VadSegmentationConfig.default
segConfig.minSpeechDuration = 0.25  // Match our 0.3s minimum
let segments = try await vadManager.segmentSpeech(audioSamples, config: segConfig)
let hasVoiceSegments = !segments.isEmpty
```

### Anti-Patterns to Avoid

- **Do NOT use SlidingWindowAsrManager for push-to-talk:** It's designed for streaming/real-time transcription. For push-to-talk, record the full utterance, then batch-transcribe via AsrManager. Batch is simpler and faster for short clips.
- **Do NOT manually parse audio samples:** Always use `AudioConverter.resampleBuffer()` or let `installTap` format conversion handle it. Manual parsing of WAV/PCM corrupts input due to metadata chunks, bit depths, and channel layouts.
- **Do NOT assume ASRResult contains language info:** Parakeet TDT v3 returns `text` and `confidence` but NO language code. Always use NLLanguageRecognizer post-hoc.
- **Do NOT block the main thread during model loading:** AsrModels.downloadAndLoad() and AsrManager.loadModels() are async operations. First-run download is ~2.69 GB; CoreML encoder compilation takes ~3.4s. Use Task.detached(priority: .utility) as in current ModelWarmupService.
- **Do NOT retain WhisperKit-specific comments:** D-03 requires zero remnants. Update all doc comments to reference FluidAudio/Parakeet, not WhisperKit/Whisper.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Audio format conversion | Manual PCM resampling/channel conversion | FluidAudio `AudioConverter` or AVAudioEngine installTap format param | Edge cases with metadata chunks, bit depths, stereo-to-mono; silent corruption if done wrong [VERIFIED: FluidAudio docs warn explicitly] |
| Voice activity detection | Simple energy threshold calculation | FluidAudio `VadManager` (Silero VAD v6) | Neural VAD is significantly more accurate than energy-based; already bundled with FluidAudio [VERIFIED: VAD docs] |
| Language detection | Character set heuristics or regex | Apple `NLLanguageRecognizer` | Statistical model trained on large text corpora; supports constraining to {de, en} set; built into macOS [VERIFIED: Apple docs] |
| Model download/caching | Custom HuggingFace download logic | FluidAudio `AsrModels.downloadAndLoad()` | Handles caching, proxy support, progress reporting; uses ModelRegistry for URL construction [VERIFIED: FluidAudio source code] |
| CoreML compilation management | Manual mlmodelc compilation steps | FluidAudio's loadModels() handles compilation | Automatic CoreML compilation/caching on first load; handles int8 vs fp32 encoder variant selection [VERIFIED: ParakeetLanguageModels.swift source] |

**Key insight:** FluidAudio handles the entire model lifecycle (download, compile, load, infer, cleanup) but explicitly does NOT handle audio capture. The app's responsibility boundary is: capture audio -> convert to [Float] at 16kHz -> pass to AsrManager.

## Common Pitfalls

### Pitfall 1: No Language Code in ASRResult
**What goes wrong:** Code expects `result.language` like WhisperKit provided; ASRResult has only `text`, `confidence`, `duration`, `processingTime`, `tokenTimings`.
**Why it happens:** Parakeet TDT v3 (FastConformer-TDT architecture) does not output language identification. The model auto-detects internally for transcription quality but exposes no language metadata. NVIDIA confirmed this in NeMo GitHub issue #15097 (closed Dec 2025): "It is not supported by the model to output language."
**How to avoid:** Use Apple's NLLanguageRecognizer on the transcribed text with `languageConstraints` set to `[.german, .english]`. Works on macOS 14+.
**Warning signs:** Compiler error on `result.language` access; empty/nil language field.
[VERIFIED: ASRResult source code in AsrTypes.swift, NVIDIA NeMo issue #15097]

### Pitfall 2: AVAudioEngine Format Mismatch
**What goes wrong:** Passing raw AVAudioEngine buffers (typically 44.1kHz/48kHz stereo) directly to AsrManager.transcribe() produces garbage or empty transcriptions.
**Why it happens:** Parakeet CoreML models require exactly 16kHz mono Float32 input. FluidAudio docs explicitly warn against manual format parsing.
**How to avoid:** Either (a) specify target format in `installTap(format:)` to get automatic conversion, or (b) use FluidAudio's `AudioConverter.resampleBuffer()` on each captured buffer.
**Warning signs:** Empty transcription results despite audible speech in recording.
[VERIFIED: FluidAudio docs "Do not parse WAV/PCM bytes by hand"]

### Pitfall 3: First-Run Model Download Blocks UI
**What goes wrong:** ~2.69 GB model download on first launch hangs the app if done on main actor.
**Why it happens:** AsrModels.downloadAndLoad() is async but the download is large. CoreML encoder compilation adds ~3.4s even after download.
**How to avoid:** Preserve the existing ModelWarmupService pattern: Task.detached(priority: .utility) with timeout watchdog. Show "Preparing models..." in the menu bar dropdown.
**Warning signs:** Spinning beachball or unresponsive menu bar on first launch.
[VERIFIED: FluidAudio benchmarks -- encoder first-load 3361ms on iPhone 16 Pro Max]

### Pitfall 4: AsrManager is an Actor -- All Calls Require Await
**What goes wrong:** Compiler errors about actor isolation when calling AsrManager methods.
**Why it happens:** Since FluidAudio v0.12.6, AsrManager uses Swift actor concurrency. All method calls require `await`. This is a breaking change from pre-0.12.6.
**How to avoid:** Store AsrManager reference and always call with `await`. Since TranscriptionService is @MainActor, calls to the AsrManager actor require suspension points (async).
**Warning signs:** "Actor-isolated property" or "Expression is 'async' but is not marked with 'await'" compiler errors.
[VERIFIED: FluidAudio v0.12.6 release notes on GitHub]

### Pitfall 5: NLLanguageRecognizer Accuracy on Short Text
**What goes wrong:** Language detection returns wrong language for single-word or very short transcriptions.
**Why it happens:** NLLanguageRecognizer uses statistical models that need sufficient text context. Single words like "ja" (German) or "the" (English) may be ambiguous.
**How to avoid:** For very short transcriptions (< 5 words), consider defaulting to "en" or using the previous detection as a hint. The `languageConstraints` property helps by limiting choices to {de, en}.
**Warning signs:** Inconsistent language codes for short phrases in mixed-language sessions.
[ASSUMED -- based on NLLanguageRecognizer documentation noting performance varies with text length]

### Pitfall 6: VadManager Model Not Loaded
**What goes wrong:** VadManager.process() throws because the Silero VAD CoreML model isn't loaded yet.
**Why it happens:** VadManager needs its own model loaded separately from the ASR models. If warmup only loads ASR models, VAD calls fail.
**How to avoid:** Initialize VadManager during ModelWarmupService.warmup() alongside AsrModels download. Silero model is tiny (~1.8 MB) so adds negligible time.
**Warning signs:** Runtime crash on first transcription attempt with "model not initialized" error.
[VERIFIED: FluidAudio VAD docs show separate initialization path]

### Complete ModelWarmupService Replacement Pattern
```swift
// Source: FluidAudio README + AsrTypes.swift + VAD docs
import SwiftUI
import FluidAudio

@MainActor
class ModelWarmupService: ObservableObject {
    @Published var isWarming = false
    @Published var isReady = false
    @Published var error: String?
    
    private var asrManager: AsrManager?
    private var vadManager: VadManager?
    private var warmupTask: Task<Void, Never>?
    private let warmupTimeoutSeconds: UInt64 = 600
    
    var showWarmupRow: Bool { isWarming || error != nil }
    var statusText: String? {
        if isWarming { return "Preparing models\u{2026}" }
        if let error { return error }
        return nil
    }
    
    func warmup() {
        guard !isWarming && !isReady else { return }
        isWarming = true
        error = nil
        
        warmupTask = Task.detached(priority: .utility) { [weak self] in
            do {
                // Step 1: Download + load Parakeet TDT v3 CoreML models
                let models = try await AsrModels.downloadAndLoad(version: .v3)
                let manager = AsrManager(config: .default)
                try await manager.loadModels(models)
                
                // Step 2: Initialize Silero VAD (tiny ~1.8 MB model)
                let vad = try await VadManager(config: VadConfig(defaultThreshold: 0.75))
                
                try Task.checkCancellation()
                
                await MainActor.run {
                    self?.asrManager = manager
                    self?.vadManager = vad
                    self?.isWarming = false
                    self?.isReady = true
                }
            } catch is CancellationError {
                await MainActor.run {
                    self?.isWarming = false
                    self?.error = "Model load timed out or was cancelled. Restart app."
                }
            } catch {
                await MainActor.run {
                    self?.isWarming = false
                    self?.error = "Model load failed. Restart app."
                }
            }
        }
        
        // Timeout watchdog
        Task { [weak self] in
            try? await Task.sleep(nanoseconds: (self?.warmupTimeoutSeconds ?? 600) * 1_000_000_000)
            guard let self else { return }
            if self.isWarming { self.cancelWarmup() }
        }
    }
    
    func cancelWarmup() {
        warmupTask?.cancel()
        warmupTask = nil
    }
    
    /// Expose initialized AsrManager for TranscriptionService
    var asrManagerInstance: AsrManager? { asrManager }
    
    /// Expose initialized VadManager for TranscriptionService
    var vadManagerInstance: VadManager? { vadManager }
}
```

### Complete TranscriptionService Replacement Pattern
```swift
// Source: FluidAudio API + Apple AVAudioEngine + NaturalLanguage docs
import SwiftUI
import FluidAudio
import AVFoundation
import NaturalLanguage

@MainActor
class TranscriptionService: ObservableObject {
    enum State: Equatable, Sendable {
        case idle, recording, transcribing
    }
    
    @Published var state: State = .idle
    @Published var lastResult: DicticusTranscriptionResult?
    @Published var error: String?
    
    var silenceThreshold: Float = 0.5  // May need recalibration for Silero VAD
    let minimumDurationSeconds: Float = 0.3
    
    private let asrManager: AsrManager
    private let vadManager: VadManager
    private let audioEngine = AVAudioEngine()
    private var audioSamples: [Float] = []
    private let sampleRate: Double = 16000
    
    init(asrManager: AsrManager, vadManager: VadManager) {
        self.asrManager = asrManager
        self.vadManager = vadManager
    }
    
    func startRecording() throws {
        guard state == .idle else { throw TranscriptionError.busy }
        audioSamples = []
        
        let inputNode = audioEngine.inputNode
        let inputFormat = inputNode.outputFormat(forBus: 0)
        
        // Target format: 16kHz mono Float32 (what Parakeet expects)
        let targetFormat = AVAudioFormat(
            commonFormat: .pcmFormatFloat32,
            sampleRate: sampleRate,
            channels: 1,
            interleaved: false
        )!
        
        // Install tap -- accumulate samples in 16kHz mono
        // Note: AVAudioEngine may not support direct format conversion via installTap
        // on all hardware; may need AVAudioConverter as fallback
        inputNode.installTap(onBus: 0, bufferSize: 4096, format: inputFormat) {
            [weak self] buffer, _ in
            guard let self else { return }
            // Convert buffer to target format if needed, then extract Float32 samples
            // This runs on the audio thread -- keep work minimal
            if let channelData = buffer.floatChannelData?[0] {
                let frameCount = Int(buffer.frameLength)
                let samples = Array(UnsafeBufferPointer(start: channelData, count: frameCount))
                // TODO: Resample from inputFormat.sampleRate to 16kHz if needed
                // Use AudioConverter.resampleBuffer() or manual resampling
                Task { @MainActor in
                    self.audioSamples.append(contentsOf: samples)
                }
            }
        }
        
        try audioEngine.start()
        state = .recording
    }
    
    func stopRecordingAndTranscribe() async throws -> DicticusTranscriptionResult {
        guard state == .recording else { throw TranscriptionError.notRecording }
        
        audioEngine.stop()
        audioEngine.inputNode.removeTap(onBus: 0)
        state = .transcribing
        defer { if state == .transcribing { state = .idle } }
        
        let samples = audioSamples
        let durationSeconds = Float(samples.count) / Float(sampleRate)
        
        // Layer 1: Minimum duration guard (D-11)
        guard durationSeconds >= minimumDurationSeconds else {
            state = .idle
            throw TranscriptionError.tooShort
        }
        
        // Layer 2: Silero VAD pre-filter (replaces WhisperKit EnergyVAD)
        let vadResults = try await vadManager.process(samples)
        let hasVoice = vadResults.contains { $0.probability > silenceThreshold }
        guard hasVoice else {
            state = .idle
            throw TranscriptionError.silenceOnly
        }
        
        // Layer 3: Transcribe via Parakeet TDT v3
        let result = try await asrManager.transcribe(samples)
        
        guard !result.text.trimmingCharacters(in: .whitespacesAndNewlines).isEmpty else {
            state = .idle
            throw TranscriptionError.noResult
        }
        
        // Post-hoc language detection (Parakeet outputs no language code)
        let detectedLanguage = detectLanguage(result.text)
        
        let transcriptionResult = DicticusTranscriptionResult(
            text: result.text.trimmingCharacters(in: .whitespacesAndNewlines),
            language: detectedLanguage,
            confidence: result.confidence
        )
        
        lastResult = transcriptionResult
        state = .idle
        return transcriptionResult
    }
    
    func detectLanguage(_ text: String) -> String {
        let recognizer = NLLanguageRecognizer()
        recognizer.languageConstraints = [.german, .english]
        recognizer.processString(text)
        guard let language = recognizer.dominantLanguage else { return "en" }
        switch language {
        case .german: return "de"
        case .english: return "en"
        default: return "en"
        }
    }
    
    func restrictLanguage(_ detected: String) -> String {
        let allowed: Set<String> = ["de", "en"]
        return allowed.contains(detected) ? detected : "en"
    }
}
```

## Open Questions (RESOLVED)

1. **Audio resampling strategy** — RESOLVED
   - What we know: AVAudioEngine input is typically 44.1/48kHz; Parakeet needs 16kHz. FluidAudio provides AudioConverter.resampleBuffer() for AVAudioPCMBuffer.
   - What's unclear: Whether installTap's format parameter can directly request 16kHz output, or whether explicit resampling via AudioConverter is always needed.
   - RESOLVED: Start with installTap at native rate, resample in stopRecordingAndTranscribe() via AVAudioConverter (primary) or resampleAudio() linear interpolation (fallback). See Plan 02 Task 1.

2. **Silero VAD threshold calibration** — RESOLVED
   - What we know: VadConfig has defaultThreshold (0.85 default). VadManager.process() returns per-chunk probabilities.
   - What's unclear: What probability threshold optimally balances false positives (noise accepted) vs false negatives (quiet speech rejected) for push-to-talk dictation.
   - RESOLVED: Use 0.5 probability threshold for Silero VAD hasVoice check. Lower than Silero default (0.85) because push-to-talk has a clear intent signal. See Plan 02 Task 1 silenceThreshold.

3. **AsrManager actor isolation with @MainActor TranscriptionService** — RESOLVED
   - What we know: AsrManager is an actor. TranscriptionService is @MainActor. Cross-actor calls require await.
   - What's unclear: Whether storing AsrManager reference in a @MainActor class causes Swift 6 isolation issues (like the nonisolated(unsafe) pattern needed for WhisperKit).
   - RESOLVED: No nonisolated(unsafe) needed. AsrManager is a proper Swift actor; cross-actor calls from @MainActor TranscriptionService work naturally with await. See Plan 02 Task 1.

4. **CoreML model cache location** — RESOLVED
   - What we know: FluidAudio uses ModelRegistry with HuggingFace URLs. Silero VAD caches at ~/Library/Application Support/FluidAudio/Models/.
   - What's unclear: Exact cache path for Parakeet models. Whether there's a way to check if models are already cached (for warm-start detection).
   - RESOLVED: Check ~/Library/Application Support/FluidAudio/Models/ for cached models. AsrModels.downloadAndLoad() handles caching internally. isFluidAudioAvailable() test helper checks this path. See Plan 02 Task 2.
