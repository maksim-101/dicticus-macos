# Phase 2.1 Plan 01: Replace WhisperKit with FluidAudio in ModelWarmupService Summary

FluidAudio 0.13.6 replaces WhisperKit 0.18.0 in project.yml and ModelWarmupService, initializing Parakeet TDT v3 CoreML via AsrModels/AsrManager and Silero VAD via VadManager while preserving the full public contract.

## What Was Built

- `project.yml`: WhisperKit SPM package replaced with `FluidAudio` from `https://github.com/FluidInference/FluidAudio.git` at `from: 0.13.6`. Target dependency updated from `WhisperKit` product to `FluidAudio` product.
- `ModelWarmupService.swift`: Fully rewritten internals — `import WhisperKit` replaced with `import FluidAudio`, `whisperKit: WhisperKit?` replaced with `asrManager: AsrManager?` + `vadManager: VadManager?`. Warmup sequence now calls `AsrModels.downloadAndLoad(version: .v3)` then `AsrManager(config: .default).loadModels(models)` then `VadManager(config: VadConfig(defaultThreshold: 0.75))`. Public contract (isWarming, isReady, error, showWarmupRow, statusText, warmup(), cancelWarmup()) preserved verbatim.
- `project.pbxproj`: Regenerated via `xcodegen generate` reflecting new FluidAudio SPM dependency.
- `ModelWarmupServiceTests.swift`: Single comment updated from "WhisperKit initialization" to "FluidAudio initialization". All 13 test methods preserved with identical assertion logic.

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| FluidAudio 0.13.6 minimum version | Matches RESEARCH.md verified version; actor-based AsrManager introduced in 0.12.6 |
| warmupTimeoutSeconds = 600 preserved | First-run download is ~2.69 GB Parakeet CoreML; 10-minute ceiling appropriate |
| VadManager initialized in same warmup() call | VadManager needs its own model loaded before first transcription (RESEARCH.md Pitfall 6); Silero model is tiny ~1.8 MB, negligible overhead |
| Task.detached + MainActor.run pattern preserved | Established project pattern; avoids blocking main actor during large download/compilation |
