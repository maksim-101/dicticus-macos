  <action>
    1. Add `case unexpectedLanguage` to `TranscriptionError` enum in TranscriptionService.swift.

    2. Add a `static func containsNonLatinScript(_ text: String) -> Bool` method to TranscriptionService. The check should:
       - Iterate over `text.unicodeScalars`
       - For each scalar, skip if it is in the ASCII range (U+0000-U+007F), Latin Extended ranges (U+0080-U+024F, U+1E00-U+1EFF, U+2C60-U+2C7F, U+A720-U+A7FF), common punctuation/symbols/numbers, or combining diacritical marks (U+0300-U+036F)
       - If any scalar falls in Cyrillic (U+0400-U+04FF), CJK (U+4E00-U+9FFF), Arabic (U+0600-U+06FF), Devanagari (U+0900-U+097F), or other non-Latin letter blocks, return true
       - The simplest correct approach: check if the scalar is a Unicode letter (`CharacterSet.letters.contains(scalar)`) AND is NOT in the Latin character sets above. This catches all non-Latin scripts without needing to enumerate every block.
       - Return false if no non-Latin letter scalars are found
       - Make it `static` so it can be unit tested without FluidAudio instances (same pattern as existing `testRestrictLanguage` and `testDetectLanguage`)

    3. In `stopRecordingAndTranscribe()`, AFTER the ASR transcription succeeds (after the `guard !result.text.trimmingCharacters(in: .whitespacesAndNewlines).isEmpty` check), and BEFORE building the `DicticusTranscriptionResult`, add:
       ```swift
       // Script validation: reject non-Latin output (Parakeet may output Cyrillic/CJK/Arabic)
       let trimmedText = result.text.trimmingCharacters(in: .whitespacesAndNewlines)
       guard !Self.containsNonLatinScript(trimmedText) else {
           throw TranscriptionError.unexpectedLanguage
       }
       ```
       Note: the `trimmedText` variable already exists from the empty check above -- reuse it or extract it once. The guard should use the already-trimmed text.

    4. Add `case unexpectedLanguage` to `DicticusNotification` enum in NotificationService.swift:
       ```swift

       ```
       With message: `"Unexpected language detected. Please try again."` -- concise, actionable, under 80 chars.

    5. In HotkeyManager.swift `handleKeyUp(mode:)`, in the `catch let error as TranscriptionError` block, add a case for `.unexpectedLanguage` that posts the notification (NOT silent like tooShort/silenceOnly -- user needs to know why text was not injected):

  <action>
    **Fix 1: Missing space between dictation segments (major)**

    In TextInjector.swift `injectText(_ text: String)`, prepend a space before the text being injected. Change step 2 from:
    ```swift
    pasteboard.setString(text, forType: .string)
    ```
    to:
    ```swift
    pasteboard.setString(" " + text, forType: .string)
    ```

    This matches the behavior of other dictation apps (MacWhisper, Speak2). The leading space ensures consecutive dictation segments don't merge. If the user dictates into an empty field, the leading space is harmless (one extra space at the start is far less disruptive than merged words mid-sentence). This is the simplest correct approach and is what the UAT diagnosis specifies.

    **Fix 2: mic.slash icon on app restart (minor)**

    In DicticusApp.swift, the `iconName` computed property checks `permissionManager.allGranted` -- if permissions haven't been polled yet, both statuses default to `.pending`, so `allGranted` is false and the icon shows `mic.slash`. The fix: call `permissionManager.checkAll()` during app init so permissions are checked at launch, not just when the dropdown opens.

    In the `label:` closure of `MenuBarExtra`, the `.task` modifier already exists (calling `warmupService.warmup()`). Add `permissionManager.checkAll()` as the FIRST line inside that `.task` closure:
    ```swift
    .task {
        permissionManager.checkAll()
        warmupService.warmup()
    }
    ```

    The `.task` modifier fires when the label view appears (which is at app launch for menu bar apps, unlike `.onAppear` in the dropdown content). This ensures permissions are polled immediately, so `iconName` reads the correct state from the start.

    **Fix 3: Copy button visual feedback (cosmetic)**

    In LastTranscriptionView.swift, add a `@State private var showCopied = false` property. Replace the Copy button with a conditional label:
    ```swift
    Button(showCopied ? "Copied!" : "Copy") {
        NSPasteboard.general.clearContents()
        NSPasteboard.general.setString(text, forType: .string)
        showCopied = true
        // Reset after 1.5 seconds
        DispatchQueue.main.asyncAfter(deadline: .now() + 1.5) {
            showCopied = false
        }
    }
    .controlSize(.small)
    .buttonStyle(.bordered)
    .accessibilityLabel(showCopied ? "Copied to clipboard" : "Copy last transcription")
    ```

    This gives immediate visual feedback without requiring animation frameworks. The label changes to "Copied!" for 1.5 seconds, then reverts. The accessibility label also updates for VoiceOver users.
