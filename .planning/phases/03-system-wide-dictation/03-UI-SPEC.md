## Color

All colors are macOS semantic system colors. No custom hex values. Adapts to light/dark mode automatically.

| Role | SwiftUI Semantic Token | Usage |
|------|----------------------|-------|
| Dominant (60%) | `.background` / `Color(NSColor.windowBackgroundColor)` | Dropdown background, hotkey config section background |
| Secondary (30%) | `Color(NSColor.controlBackgroundColor)` | Last transcription preview background |
| Accent (10%) | `Color.accentColor` | "Copy" button in last transcription row only |
| Recording | `Color.red` / `Color(NSColor.systemRed)` | Menu bar icon tint while recording (D-09), `mic.fill` red tint |
| Destructive | `Color.red` | Not used in Phase 3 — no destructive actions |
| Success | `Color.green` | Permission rows carried from Phase 1 — not new in Phase 3 |

Accent reserved for:
- "Copy" button in the last transcription preview row (only interactive CTA new in Phase 3)

Not used for: decorative elements, hotkey recorder views (those use system default KeyboardShortcuts styling), recording indicator (uses `.red` semantic color, not `.accentColor`), icon states other than the Copy CTA.

## Menu Bar Icon States

Extends Phase 1 icon states. Driven by `TranscriptionService.State` enum.

| State | SF Symbol | Rendering | Source |
|-------|-----------|-----------|--------|
| Idle / ready | `mic` | Template image, system tint | Phase 1 UI-SPEC |
| Warming up | `mic` + `.symbolEffect(.pulse)` | Pulsing animation | Phase 1 UI-SPEC (carried) |
| Permission missing | `mic.slash` | Template image, monochrome | Phase 1 UI-SPEC (carried) |
| **Recording (new)** | `mic.fill` | `.foregroundStyle(.red)` — filled red mic | CONTEXT.md D-09 |
| **Transcribing (new)** | `waveform.circle` | `.foregroundStyle(.secondary)` + `.symbolEffect(.pulse)` | Claude's discretion — see rationale below |

**Transcribing state SF Symbol rationale (Claude's discretion):**
`waveform.circle` is chosen over `ellipsis.circle` (mentioned in D-11) because it semantically represents audio/speech processing rather than generic "waiting." It is immediately distinguishable from the `ProgressView` spinner already used in the WarmupRow, avoids confusion with the iOS ellipsis meaning "more options," and conveys that the waveform is being analyzed. Use `.symbolEffect(.pulse)` to indicate ongoing work, consistent with the warm-up animation pattern established in Phase 1.

Source: CONTEXT.md D-09 (recording state), D-11 (transcribing state, symbol at Claude's discretion); Phase 1 UI-SPEC (.symbolEffect pattern)

## Dropdown Menu Structure (Phase 3)

Extends Phase 1 structure. New sections added above the Quit button.

```
[Menu Bar Icon — mic / mic.fill (red) / waveform.circle (pulsing) / mic.slash]
├── [Warm-up row — hidden after models are ready, carried from Phase 1]
│   ProgressView  "Preparing models…"
├── Divider (conditional — only if warm-up row is visible)
├── [Permission rows — always visible, carried from Phase 1]
│   🟢/🔴/🕐  Microphone        [Grant Access / Open Settings]
│   🟢/🔴/🕐  Accessibility      [Grant Access / Open Settings]
│   🟢/🔴/🕐  Input Monitoring   [Grant Access / Open Settings]
├── Divider
├── [Hotkey Config section — new in Phase 3, always visible]
│   Section heading: "Hotkeys"  (.headline, semibold)
│   KeyboardShortcuts.Recorder  "Plain Dictation"
│   KeyboardShortcuts.Recorder  "AI Cleanup"
├── Divider
├── [Last Transcription section — new in Phase 3, hidden when no transcription yet]
│   Section heading: "Last Transcription"  (.headline, semibold)
│   Text (2-line truncated, .body)   [Copy]
├── Divider
└── Quit Dicticus
```

Layout rules:
- Hotkey section heading: `Text("Hotkeys").font(.headline)` left-aligned, `padding(.horizontal)`, `padding(.top, 4)`
- `KeyboardShortcuts.Recorder` rows: label left, recorder control right, same `.padding(.horizontal).padding(.vertical, 4)` as PermissionRow
- Last transcription section heading: `Text("Last Transcription").font(.headline)` same pattern as Hotkeys heading
- Last transcription preview: `.lineLimit(2)` on body text, `.truncationMode(.tail)`, left-aligned
- Copy button: `.controlSize(.small)`, `.buttonStyle(.bordered)`, right-aligned in same HStack row
- Last transcription section hidden entirely when no transcription has occurred in current session

Source: CONTEXT.md D-20/D-21/D-22; Phase 1 UI-SPEC (PermissionRow spacing pattern applied to new rows)

## Interaction States

### Menu Bar Icon Transition Sequence

Full state machine for the icon driven by TranscriptionService.State:

```
App launch
  → warming up:    mic (pulsing)
  → models ready:  mic (static)
  → key held:      mic.fill (red, static)
  → key released:  waveform.circle (pulsing) — transcribing
  → paste done:    mic (static)
  → key held again: mic.fill (red, static)
```

Source: CONTEXT.md D-09/D-11 (states); Phase 1 UI-SPEC (.symbolEffect pattern)

### Hotkey Config Rows

| State | Visual | Interaction |
|-------|--------|-------------|
| Default (not customized) | Recorder shows default combo (Control+Shift+S / Control+Shift+D) | Click recorder → enter new combo |
| User-customized | Recorder shows user's combo | Click recorder → enter new combo |
| AI Cleanup row | Always visible; recorder functional even though stub in Phase 3 | User can assign a hotkey; it is silently ignored until Phase 4 |

Notes: KeyboardShortcuts.Recorder handles conflict detection, display formatting, and clear/reset natively. Do not add custom validation UI.

Source: CONTEXT.md D-12/D-13/D-14; RESEARCH.md §Standard Stack (KeyboardShortcuts features)

### Last Transcription Preview Row

| State | Visual |
|-------|--------|
| No transcription yet (session start) | Section hidden entirely — no empty state placeholder |
| Transcription available | 2-line truncated text (`.body`, `.primary`) + "Copy" button (`.controlSize(.small)`, `.buttonStyle(.bordered)`) |
| After copy button pressed | No visual feedback beyond system behavior — button remains, text unchanged |

Truncation: `.lineLimit(2)` + `.truncationMode(.tail)`. No character count limit on the stored value — truncation is visual only.

Source: CONTEXT.md D-21 (last transcription with copy button, truncated); Claude's discretion for truncation depth and section hide behavior

### Notification States (system notifications, not in-app)

| Trigger | Notification title | Notification body |
|---------|--------------------|-------------------|
| Hotkey pressed while model still loading | "Dicticus" | "Models still loading, please wait a moment." |
| Hotkey pressed while already transcribing | "Dicticus" | "Still processing — try again in a moment." |
| Transcription failed (model error) | "Dicticus" | "Transcription failed. Check that models are loaded." |
| Recording failed (mic unavailable) | "Dicticus" | "Could not start recording. Check microphone permission." |

Rules:
- All notifications use app name "Dicticus" as title — single source of identity
- Body copy: problem statement first, then what to do, under 80 characters
- No notifications for: silence-only recording (D-16), short press discards (D-02), successful transcription (silent operation per D-10)
- Notification delivery: immediate (trigger: nil in UNNotificationRequest)

Source: CONTEXT.md D-15/D-16/D-17/D-19; RESEARCH.md §Code Examples (UNUserNotificationCenter); Claude's discretion for exact wording
