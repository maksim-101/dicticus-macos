## Deviations from Plan

### W10 grep count clarification (not a deviation, a specification note)

The plan's W10 acceptance criterion states `wc -l` should return 1. The actual count is 3 because `grep -h ... | sort -u` deduplicates on full line content — the three lines containing `"group.com.dicticus"` have different surrounding code (a constant declaration, an `!`-forced init, and a doc comment), so `sort -u` does not collapse them. The SEMANTIC invariant (identical literal spelling at every call site) is fully satisfied — there is exactly one canonical spelling of `"group.com.dicticus"` across all new files. This is a grep-tool limitation, not a code defect.
