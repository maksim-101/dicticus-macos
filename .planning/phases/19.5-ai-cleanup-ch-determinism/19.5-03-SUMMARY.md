## Design Decisions Taken

### SwissHelvetisms (D-D2 + W9)

- **30 hand-curated entries** from German Wikipedia "Liste von Helvetismen" (CC BY-SA 3.0). Daily-use frequency prioritised over exhaustiveness — covers Velo/Trottoir/parkieren/Billett/Perron/Camion/Poulet/Rüebli/Cervelat/Znacht/Zmorgen/Zvieri/Estrich/Lavabo/Cheminée/Detailhandel/Identitätskarte/Matur/Ferien/allfällig/zuhanden/anfangs/grillieren/Spital/Tram/Nachtessen/Morgenessen/Kantonsschule/Abwart/Rande.
- **W9 fix** — public `licenseAttribution` constant exposes the attribution string for runtime test assertions, so Plan 05 tests don't need to walk `#filePath` to verify the source-file header. Constant contains both required substrings ("CC BY-SA 3.0" and "Liste von Helvetismen").
- **Public enum namespace** (not class/struct) matches the `DictationMode.swift` convention for constants-only modules.

### CurrencyAntiFlip (D-B1a + D-B1c)

- **Token-walk strategy via NSRegularExpression** (not Swift `Regex`) — matches the codebase's established CleanupService.stripPreamble idiom. Pattern is bounded literal-alternation: `\b(CHF|Franken|Fr\.|Rappen|EUR|Euro|€|USD|Dollar|\$|GBP|Pfund|£)\b`. No catastrophic backtracking risk.
- **Reverse iteration in revertCurrencyFlip** — `enumerated().reversed()` so earlier-index replacements don't shift the ranges of later tokens.
- **Conservative bail-out on count mismatch** — if input and output detected currency counts differ, return output unchanged. Prevents wrong-currency substitution when the LLM has heavily rewritten the sentence (T-19.5-03-05 mitigation).
- **Canonical-label mirroring** — when reverting, the replacement label visually matches the OUTPUT token's case (3-letter all-caps code → use code; spelled form → use spelled form).
- **No state, no instances** — pure static functions, never throws, returns original input on any failure (graceful-degradation contract D-26).

### SwissNumberFormatter (D-C1 + D-C2 + D-C3 + B3 + W7)

- **Whitespace tokenization** — token-by-token walk preserving spacing, mirrors `ITNUtility.applyEnglishITN`.
- **B3 parser-ordering fix** — pre-classify token by punctuation BEFORE parsing:
  - Token contains `'` or U+2019 → already Swiss-formatted, parseSwiss only.
  - Otherwise → parseGerman first (covers `1.250` as 1250), fall back to parseSwiss as safety net.
  - Avoids the silent-corruption bug where `1.250` would parse as `1.25` under en_US_POSIX.
- **W7 currency-glyph fix** — strip leading €/$/£ before parsing, re-attach on output. `€6,70` → `€6.70`.
- **Trailing punctuation stripping** — `.,;:?!` detached before parse, re-attached on output.
- **Explicit separator assignment** — `formatter.groupingSeparator = "'"` (ASCII apostrophe U+0027) and `formatter.decimalSeparator = "."` are explicit settings, not relying on locale defaults.
- **U+2019 belt-and-suspenders** — `.replacingOccurrences(of: "\u{2019}", with: "'")` after the formatter call defends against future iOS NumberFormatter behavior changes.
- **fractionDigitCount heuristic** — preserves input precision: `5.70` stays `5.70`, but `1.250` (German thousands, 3 digits after dot, no other separator) emits with 0 fraction digits → `1'250`.
