## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19.5-04-01 | Tampering / wrong-output | `revertCurrencyFlip` corrupting valid output | mitigate | The utility's graceful-degradation contract (Plan 03) ensures unexpected shapes return the input unchanged. The revert only fires when input/output token counts match AND families differ — both conservative gates. Plan 06 fixtures cover positive (legitimate flip), negative (no flip needed), and degenerate (count mismatch) cases. |
| T-19.5-04-02 | DoS | Helvetism block bloating prompt context | accept | ~30 words at ~6 chars avg = ~180 chars added to prompt. Gemma's 2048-token context (D-05) is well above this; no measurable inference-time cost. |
| T-19.5-04-03 | Information disclosure | New log lines leaking transcript content | mitigate | No new `log.info`/`log.error` calls introduced in either edit. The existing `log.info("After strip (...)..., \(cleaned.prefix(500), privacy: .public)")` line is unchanged — same exposure as before, no expansion. |
| T-19.5-04-04 | Spoofing | LLM-injected fake STRICT/HELVETISMS block confusing the model | accept | The `sanitizeControlTokens` step (already in `build`) strips Gemma turn tokens. STRICT/HELVETISMS are plain text inside our prompt; the LLM cannot inject them into its own output to confuse a downstream consumer because there is no downstream consumer parsing for these tags. |
