# Phase 19.5: AI Cleanup CH-Determinism — Context

**Gathered:** 2026-04-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Make AI-cleanup output reliably **Swiss-correct on both iOS and macOS** in a deterministic way that doesn't rely on Gemma's prompt obedience. Two independent buckets:

1. **CH-determinism (the polish layer):** Currency formatting (CHF/EUR), apostrophe thousands separator, decimal point, Helvetism preservation, Bundeskanzlei orthography in the LLM prompt — all governed by the existing `useSwissGerman` AppGroup toggle, plus a German-locale anti-flip mechanism that protects the currency LABEL even when the toggle is off.
2. **B2 hotfix (the regression):** False "Parakeet ASR not downloaded" prompt on iOS app relaunch — folded in because it blocks reliable UAT of the cleanup features.

Per S7 (cross-platform decision), both buckets land on **iOS AND macOS in this phase**. Implementation lives in `Shared/` where it can; per-platform glue (the new macOS Swiss-toggle UI row, iOS Settings text) is the only platform-specific code. Net effect: 19.5 is the canonical Swiss output pass for both Apple platforms, replacing whatever ad-hoc / partial behavior currently exists.

**Out of scope:**
- Any new AI cleanup capability beyond what 19 shipped (no structural rewrites, no per-utterance mode toggles, no model swap).
- Any iOS UX polish (button sizes, scroll, history) — that's 19.6.
- Any macOS hygiene work (permissions, install) — that shipped in 19.7.
- Schwyzerdütsch dialect ASR — out for v2.1, separate research milestone.
- Bundeskanzlei STYLE prompt expansion beyond the rules already proven necessary by UAT (deeper grammar rules can come in a future polish phase).

</domain>

<decisions>
## Implementation Decisions

### A. Swiss German toggle: default & macOS UI

- **D-A1:** **Flip the Swiss German toggle default to ON.** Reverses Phase 19 D-15 (which defaulted OFF). Reconciles `feedback_swiss_german_default.md` (memory: "default ON, manual override allowed, NO geolocation auto-detect"), S1 in `19-UAT-FINDINGS.md`, and the user-as-primary-segment reasoning. Manual override remains the only way to disable; no auto-detection of any kind.
- **D-A2:** **The Swiss German toggle is exposed on macOS in the menu-bar dropdown** (`MenuBarView.swift`) as a Settings-style row placed next to `AiCleanupInfoView`. iOS keeps its existing `AiCleanupSection.swift` Settings location. Both bind to the same `useSwissGerman` key in the `group.com.dicticus` `UserDefaults` suite.
- **D-A3:** **One-time migration flips ALL existing installs to ON.** On first launch of v2.1, if the `useSwissGerman` key is unset (object is nil, not just `false`), set it explicitly to `true`. Use a separate idempotent flag (`swissDefaultMigratedV2_1`) so the migration runs exactly once and respects users who later flip the toggle back to OFF. Applies on both iOS and macOS, scoped to the AppGroup defaults so the migration is single-source-of-truth.

### B. Currency anti-flip (B5)

- **D-B1:** **Belt-and-suspenders anti-flip pipeline.** Three coordinated mechanisms:
  - **(B1a) Pre-LLM currency detection** in `Shared/Utilities/` (new `CurrencyDetector` or extension of `ITNUtility`): scan input for tokens matching `(CHF|Franken|Fr\.|Rappen|EUR|Euro|€|USD|Dollar|\$|GBP|Pfund|£)`. Result: a list of detected currency tokens with their original text + position.
  - **(B1b) STRICT prompt anchor** in `CleanupPrompt.build`: when any currency token is detected (and `language == "de"` per D-B2), append a line: `STRICT: Keep currency exactly as written ("CHF", "Franken", "Euro", "EUR" — whichever appears in the input). Do NOT translate, convert, or substitute one currency for another.` Helvetism block (D-D2) reinforces the same.
  - **(B1c) Post-LLM diff revert.** After cleanup returns, re-detect currencies in the output. If the set of currency families differs (e.g., input had Franken/CHF but output has only Euro/EUR), substitute the model's currency tokens back to the original family using a positional best-match revert. Numeric value and surrounding format stay as the model wrote them — only the currency LABEL is corrected.
- **D-B2:** **Anti-flip fires whenever `language == "de"`, regardless of the Swiss German toggle.** The bug is a German-language Gemma bias (model "corrects" CHF→EUR for any German number-currency context), not a Swiss-only concern. Standard-German users dictating in Switzerland or quoting CHF still get protection. English (`language == "en"`) is excluded — no evidence of cross-currency flipping there and no need to add complexity.

### C. Number-formatting micro-rules

- **D-C1:** **Thousands separator is ASCII straight apostrophe `'`, not U+2019 `’`.** Apple's `NumberFormatter` for `Locale("de_CH")` emits U+2019 by default; we post-process with `.replacingOccurrences(of: "\u{2019}", with: "'")`. Rationale: matches the roadmap row notation (`1'250`), what users type, terminal/code-friendly. Bundeskanzlei accepts both forms, so this is a presentation choice with no correctness loss.
- **D-C2:** **When Swiss toggle is ON, all decimal numbers render with period as the decimal separator** — not just currency amounts. So `CHF 5.70`, `EUR 6.70`, AND `3.14 Meter` all use `.`. Implementation: the post-LLM number pass runs `NumberFormatter` (configured with `Locale("de_CH")` + `numberStyle = .decimal` for non-currency tokens) over every numeric token in the cleaned output. Trade-off accepted: diverges from the Bundeskanzlei rule that comma stays for non-currency; user explicitly stated "should always be a point" during UAT (S8 origin). Simpler implementation, single pass, single mental model.
- **D-C3 (derived from D-C1+D-C2):** **The number-formatting pass replaces D-20 of Phase 19.** D-20 left thousands separators to the LLM only; we now do them deterministically post-LLM via `NumberFormatter` with the ASCII-glyph swap. The Phase 19 D-18 Swiss prompt line ("Use Swiss thousands separator style (e.g. 1'250, not 1.250)") stays as a hint, but the deterministic pass is the source of truth.

### D. B2 hotfix + Helvetism list

- **D-D1:** **B2 (Parakeet false re-download prompt) is folded into the 19.5 plan as Plan task 1**, not split off and not run through `/gsd-debug` as a separate session. Suspected root cause sits in `iOS/Dicticus/Services/IOSModelWarmupService.swift:checkHasModels()` — `hasModels` initializes to `false` on a fresh `IOSModelWarmupService` instance, and the UI may render the "needs download" state on relaunch before `checkHasModels` runs synchronously (or runs at all on the relevant lifecycle event). Planner's task 1 is: read the lifecycle, confirm or refute the suspected race, and apply the fix (likely making `checkHasModels` run synchronously in `init()` and/or wiring it to the `scenePhase` `.active` transition). If investigation reveals the cause is inside FluidAudio rather than our code, planner escalates to a follow-up debug session — but defaulting to "fix in our code" because the UI clearly already "figures out it's cached" eventually, which means our state ends up correct, just late.
- **D-D2:** **Helvetism word list is a hardcoded constant** in a new file `Shared/Models/SwissHelvetisms.swift`. Static Swift array of ~30 high-frequency Helvetisms (Velo, Trottoir, parkieren, Billett, Perron, Camion, Poulet, Rüebli, Cervelat, Znacht, Zmorgen, Estrich, Lavabo, Detailhandel, Identitätskarte, Matur, Ferien, allfällig, zuhanden, anfangs, grillieren, Spital, Tram, …) sourced from German Wikipedia "Liste von Helvetismen" (CC BY-SA 3.0). File header includes attribution. List is loaded once and embedded in the prompt as a `HELVETISMS:` block when `useSwissGerman` is ON AND `language == "de"`. No JSON, no App Group storage, no user-editable UI in v2.1 — those are explicit deferrals.

### Claude's Discretion

- Exact regex patterns for currency detection (D-B1a) and the positional best-match revert algorithm (D-B1c) — planner/research can tighten against UAT fixtures.
- Whether the anti-flip pipeline lives in a new `CurrencyAntiFlip.swift` utility, an extension of `ITNUtility`, or methods on `CleanupPrompt` / `CleanupService` — pick whichever produces the cleanest seam given the existing `Shared/` shape.
- Wording of the STRICT prompt anchor (D-B1b) — keep concise, German if needed.
- The exact ~30 Helvetisms (D-D2) — selection from the Wikipedia list; aim for daily-use frequency, not exhaustiveness. Document the attribution string in the source-file header.
- Whether to expose `swissDefaultMigratedV2_1` as a debug-only override (e.g., for QA to re-trigger the migration on a test device) — nice-to-have, not required.
- Copy for the macOS Swiss-toggle row (D-A2) — match existing dropdown tone.

### Folded Todos

None — no pending todos in `.planning/todos/` matched 19.5 scope at the time of context capture.

## Existing Code Insights

### Reusable Assets
- **`Shared/Models/CleanupPrompt.swift`** — `build()` already has a conditional Swiss STYLE line gated on `useSwissGerman` AppGroup key + `language == "de"` (D-18). The Helvetism block (D-D2) and STRICT currency anchor (D-B1b) extend this same conditional structure. Existing `sanitizeControlTokens` and `containsMixedLanguages` stay unchanged.
- **`Shared/Utilities/ITNUtility.swift`** — `applySwissITN` (ß → ss / ẞ → SS) is the established home for rule-based Swiss text transforms. The new currency detector and number-formatter pass should live alongside it (or in a sibling utility) — same pattern: deterministic, sub-millisecond, well-tested via `SwissGerman.fixtures.json`.
- **`Shared/Services/CleanupService.swift`** — Post-LLM Swiss safety-net block already exists (~line 219–224) with the same AppGroup read pattern. The post-LLM diff revert (D-B1c) and decimal-pass (D-C2) hook in here, before/after the existing `applySwissITN` call.
- **`iOS/Dicticus/Settings/AiCleanupSection.swift`** — Existing Swiss-toggle UI with `@AppStorage` binding. Pattern to copy when adding the macOS row in `MenuBarView`.
- **AppGroup `group.com.dicticus`** — Already wired on both platforms (macOS `Dicticus.entitlements:9`, iOS App Group). The shared toggle key `useSwissGerman` is read from this suite by all four call sites: `TextProcessingService:40`, `CleanupPrompt:42`, `CleanupService:222`, and (TBD by planner) the macOS toggle UI.

### Established Patterns
- **AppGroup-scoped Swiss state:** all reads use `UserDefaults(suiteName: "group.com.dicticus") ?? UserDefaults.standard`. New macOS toggle UI must use the same pattern (NOT raw `@AppStorage` against the standard suite — that would silently desync from `Shared/`).
- **Two orthogonal toggles:** AI Cleanup × Swiss German — Phase 19 D-15 set the contract; we honor it. The default flip (D-A1) only changes Swiss; AI Cleanup stays default OFF.
- **Graceful degradation:** any cleanup failure returns raw ASR text (Phase 19 D-26). The new anti-flip pipeline must follow this — if revert logic encounters an unexpected shape, return the LLM's output unmodified rather than crashing.
- **Deterministic pre/post passes around the LLM:** the existing pipeline (`ASR → Dictionary → ITN → Swiss-ITN → [LLM Cleanup → Swiss-ITN safety-net] → Injection`) is the right shape for adding currency detection (pre) and revert + number formatter (post).
- **Test fixtures live in JSON** (`SwissGerman.fixtures.json`) and feed into XCTest; extend that file rather than introducing a new fixture system.

### Integration Points
- **`Shared/` is cross-platform** per CLAUDE.md "Multi-Platform Strategy". All new logic for D-B1, D-C2, D-C3, D-D2 lives in `Shared/`. Only the macOS toggle UI (D-A2) and iOS Settings tweaks (D-A1 default flip) are platform-specific.
- **macOS `CleanupService.swift` duplication risk:** there's both `Shared/Services/CleanupService.swift` and `macOS/Dicticus/Services/CleanupService.swift` in the tree. Planner must determine which is the live macOS path and either consolidate (preferred) or apply changes to both consistently. Phase 19 left this as Claude's Discretion; 19.5 forces a decision because the new logic must run on both platforms.
- **B2 fix site:** `IOSModelWarmupService.checkHasModels()` and the `scenePhase` / launch lifecycle in `DicticusApp.swift`. `hasModels` is the published flag that drives the Settings UI's "needs download" state.
- **Migration code site (D-A3):** `DicticusApp.swift` on each platform. Run before any service that reads `useSwissGerman`. Idempotent via `swissDefaultMigratedV2_1` flag in the same AppGroup suite.

### Constraints discovered
- **`useSwissGerman` AppGroup key has no current macOS UI** — fresh macOS users had no way to enable Swiss before this phase. The migration (D-A3) is what flips them on.
- **Apple `NumberFormatter` with `Locale("de_CH")`** emits U+2019 thousands separator. Our ASCII-only contract (D-C1) requires post-processing.
- **Gemma 4 E2B is small (~3 GB)** — small models are more sensitive to prompt nudges (helps anchor, D-B1b) but also more likely to ignore them under unusual phrasings (justifies the post-LLM revert, D-B1c).
- **No Schwyzerdütsch / dialect handling** — Parakeet ASR transcribes spoken Schwyzerdütsch as best-effort; we don't try to interpret it as standard German. Out of scope, separate research milestone.
- **No third-party Swiss German libraries** — ruled out by S4 (Apple-native) and license analysis (LanguageTool LGPL too heavy, Hunspell de_CH GPL incompatible).

<specifics>
## Specific Ideas

- **B5 user UAT quote:** "Swiss francs / Schweizer Franken transcribes to Euro in cleaned output." This is a label-substitution bug, not a number-format bug — D-B1's three-mechanism approach addresses it head-on.
- **B6 user UAT quote:** "EUR shows `6,70 Euro`, CHF shows `5.70 Franken`. Should always be a point." This drove D-C2's universal-period decision (not just currency).
- **Roadmap row uses ASCII apostrophe:** `1'250` — that's the canonical user-facing notation. D-C1 honors it.
- **Migration-flag naming:** `swissDefaultMigratedV2_1` (in the AppGroup suite). One key per migration, never reused — if v2.2 needs another locale flip, that's `swissDefaultMigratedV2_2`.
- **Belt-and-suspenders is intentional:** the user explicitly chose the three-mechanism approach over either prompt-only or post-only. Don't drop one mechanism in plan execution to "simplify" — they cover different failure modes.
- **Anti-flip is German-wide, not Swiss-only:** any German-language dictation gets currency-label protection (D-B2). Swiss-toggle gates the *number formatting* (D-C2) and the *Helvetism block* (D-D2), but currency-flip protection is broader.

</specifics>
