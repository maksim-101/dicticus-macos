```swift
@MainActor
class TextProcessingService: ObservableObject {
    init(dictionaryService: DictionaryService = .shared,
         cleanupService: CleanupProvider?,
         historyService: HistoryService = .shared)
    func process(text: String, language: String, mode: DictationMode, confidence: Double = 1.0) async -> String
}
```

`TextProcessingService.process(...)` internally:
- Applies dictionary replacements (Step 1)
- Applies rule-based ITN (Step 2)
- Applies Swiss ITN if `useSwissGerman` toggle ON (Step 2b — added in Plan 02)
- If `mode == .aiCleanup && cleanupService?.isLoaded`: runs LLM cleanup (Step 3)
- Saves `TranscriptionEntry` to History (Step 4)
- Returns processed text

    ```swift
    func stopDictation() async {
        guard state == .recording else { return }
        state = .transcribing

        do {
            guard let result = try await transcriptionService?.stopRecordingAndTranscribe() else {
                await endLiveActivity()
                state = .idle
                return
            }

            // Determine mode: AI cleanup only if toggle ON AND LLM successfully warmed.
            let appGroupDefaults = UserDefaults(suiteName: "group.com.dicticus") ?? UserDefaults.standard
            let wantsAiCleanup = appGroupDefaults.bool(forKey: "aiCleanupEnabled")
            let llmReady = cleanupService?.isLoaded ?? false
            let mode: DictationMode = (wantsAiCleanup && llmReady) ? .aiCleanup : .plain

            // Route through TextProcessingService (Dictionary -> ITN -> Swiss ITN -> [LLM] -> History).
            // TextProcessingService.process() itself saves the TranscriptionEntry (Step 4) — do NOT double-save here.
            let processor = TextProcessingService(cleanupService: cleanupService)
            let cleaned = await processor.process(
                text: result.text,
                language: result.language,
                mode: mode,
                confidence: Double(result.confidence)
            )

            UIPasteboard.general.string = cleaned
            lastResult = cleaned
            error = nil

        } catch let transcriptionError as TranscriptionError {
            switch transcriptionError {
            case .tooShort:
                self.error = "Recording too short."
            case .silenceOnly:
                self.error = "No speech detected."
            case .noResult:
                self.error = "Could not understand audio."
            case .unexpectedLanguage:
                self.error = "Unsupported language detected."
            case .modelNotReady:
                self.error = "Model not ready."
            case .busy:
                self.error = "System busy."
            case .notRecording:
                self.error = "Not recording."
            }
        } catch {
            self.error = error.localizedDescription
        }

        await endLiveActivity()
        state = .idle
    }
    ```

The new code removes that block because `TextProcessingService.process(...)` saves its own entry internally (see `Shared/Services/TextProcessingService.swift` lines 53-60). <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ASR output text → LLM prompt | Sanitized via existing `CleanupPrompt.sanitizeControlTokens` (D-27). No change in this plan. |
| DictationViewModel ↔ AppGroup UserDefaults | Reads `aiCleanupEnabled` to select mode. Same trust model as existing `useCustomDictionary` / `useITN` reads. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-06-01 | T (Tampering) / E (Elevation) | Prompt injection from malicious speech input | mitigate | `CleanupPrompt.sanitizeControlTokens` strips Gemma control tokens before interpolation (D-27, preserved). Even if an attacker speaks "`<end_of_turn>`…" the sanitizer removes the literal. Swiss prompt extension (D-18) is a static instruction appended before user text — cannot be escaped from user text alone. |
| T-19-06-02 | I (Info Disclosure) | Double-save of history leaking transcripts to disk twice | mitigate | `DictationViewModel` removes its direct `HistoryService.shared.save()` call because `TextProcessingService.process()` saves internally (Step 4 of the pipeline). Preventing double-save avoids storage bloat and duplicate entries in user-visible history. |
| T-19-06-03 | D (Denial of Service) | Blocked UI during LLM cleanup (D-13) | accept | Per D-13, dictation blocks until cleanup completes. 8s timeout (D-04) bounds worst case. User experience is "stop talking, wait up to 8s, see cleaned text". Raw-then-replace UX was explicitly rejected in v2.1 deferred ideas. |
| T-19-06-04 | E | LLM returns text containing unexpected control characters | accept | The `stripPreamble` pass + clipboard's own sanitization handles most cases. Absolute worst case: user sees unexpected text and retries. No security boundary crossed (on-device, returned to user's own clipboard). |
</threat_model>
