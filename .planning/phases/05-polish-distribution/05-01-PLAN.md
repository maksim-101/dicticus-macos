   - Import CoreGraphics and Foundation.
   - Class `ModifierHotkeyListener` (NOT @MainActor -- runs on background CFRunLoop thread).
   - Published/observable properties:
     - `plainDictationCombo: ModifierCombo` (default: `.fnShift` per D-09) -- stored in `@AppStorage("modifierPlainDictation")` or UserDefaults

     - `eventTap: CFMachPort?`
     - `runLoopSource: CFRunLoopSource?`
     - `runLoop: CFRunLoop?`
     - `previousFlags: CGEventFlags = []`
   - Static C-compatible callback `callback: CGEventTapCallBack`:
     - Guard `type == .flagsChanged`, otherwise return event unchanged.
     - Extract `ModifierHotkeyListener` from userInfo via `Unmanaged.fromOpaque`.
     - Get `currentFlags = event.flags`.
     - Call `detectTransition(from: previousFlags, to: currentFlags, plainCombo: plainDictationCombo, cleanupCombo: cleanupCombo)`.
     - Update `previousFlags = currentFlags`.
     - If transition detected, dispatch to main actor to call `onComboActivated` or `onComboReleased` closure.
     - Return `Unmanaged.passUnretained(event)`.
   - Closures for communication with HotkeyManager (set by HotkeyManager during wiring):
     - `var onComboActivated: ((DictationMode) -> Void)?`
     - `var onComboReleased: ((DictationMode) -> Void)?`
   - Static method `detectTransition(from previous: CGEventFlags, to current: CGEventFlags, plainCombo: ModifierCombo, cleanupCombo: ModifierCombo) -> (mode: DictationMode, isPress: Bool)?`:
     - Define relevant mask: `[.maskSecondaryFn, .maskShift, .maskControl, .maskAlternate]`.
     - Filter both previous and current flags to only relevant modifiers.
     - For each combo (plain, cleanup):
       - Get the combo's `flags` from ModifierCombo.
       - Check if combo was NOT fully present in previous but IS fully present in current (activation).
       - Check if combo WAS fully present in previous but is NOT fully present in current (release).
       - For activation: also check that current flags filtered to relevant mask equals EXACTLY the combo flags (no extra modifiers held). This prevents Fn+Shift+Control from triggering fnShift.
     - Return the first matching transition, or nil.
   - `func start()`:
     - Create CGEventTap with `.cgSessionEventTap`, `.headInsertEventTap`, `.listenOnly`, eventMask `1 << CGEventType.flagsChanged.rawValue`.
     - Pass `self` as userInfo via `Unmanaged.passUnretained(self).toOpaque()`.
     - If tap creation fails (no Accessibility permission), return silently.
     - Create CFRunLoopSource from the tap.
     - Detach a new thread, get its CFRunLoop, add the source to `.commonModes`, enable the tap, run the CFRunLoop.
     - Store references to eventTap, runLoopSource, runLoop.
   - `func stop()`:
     - Disable the tap via `CGEvent.tapEnable(tap:enable:false)`.
     - Stop the run loop via `CFRunLoopStop`.
     - Set references to nil.
   - Handle the `.tapDisabledByTimeout` event type in the callback: re-enable the tap via `CGEvent.tapEnable(tap:enable:true)` and return event. This handles macOS automatically disabling the tap if the callback takes too long.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CGEventTap callback | System-level event tap receives all flagsChanged events globally |
| UserInfo pointer | Unmanaged pointer passed to C callback for accessing Swift instance state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | ModifierHotkeyListener CGEventTap | mitigate | Use `.listenOnly` option (not `.defaultTap`); only listen for `.flagsChanged` events via event mask -- never sees keystrokes, only modifier state changes. No keystroke data is logged or stored. |
| T-05-02 | Elevation of Privilege | CGEventTap Accessibility permission | accept | CGEventTap requires Accessibility permission which is already managed by Phase 1 PermissionManager. If not granted, `CGEvent.tapCreate` returns nil and the listener silently does not start. No escalation path. |
| T-05-03 | Denial of Service | CGEventTap callback blocking | mitigate | Callback performs only flag comparison (O(1) pure function), then dispatches to main actor asynchronously. No blocking work in the callback. Handle `.tapDisabledByTimeout` by re-enabling the tap. |
</threat_model>
