<task type="auto">
  <name>Configure Keyboard Target in project.yml</name>
  <files>iOS/project.yml, iOS/DicticusKeyboard/DicticusKeyboard.entitlements</files>
  <action>
    1. Create `iOS/DicticusKeyboard/DicticusKeyboard.entitlements` with `group.com.dicticus` App Group.
    2. Add `DicticusKeyboard` target to `iOS/project.yml`:
       - Type: `app-extension`
       - Platform: `iOS`
       - Sources: `DicticusKeyboard`, `../Shared`
       - Entitlements: `iOS/DicticusKeyboard/DicticusKeyboard.entitlements`
       - Settings: `PRODUCT_BUNDLE_IDENTIFIER: com.dicticus.ios.keyboard`, `INFOPLIST_FILE: DicticusKeyboard/Info.plist`, `GENERATE_INFOPLIST_FILE: YES`
       - Info:
         - `NSExtensionPointIdentifier: com.apple.keyboard-service`
         - `NSExtensionPrincipalClass: DicticusKeyboard.KeyboardViewController`
         - `RequestsOpenAccess: true`
    3. Update `Dicticus` application target to depend on `DicticusKeyboard`.
    4. Run `xcodegen` in `iOS/` directory to update the `.xcodeproj`.
  </action>
  <verify>
    <automated>cd iOS && xcodegen && xcodebuild -list -project Dicticus.xcodeproj | grep DicticusKeyboard</automated>
  </verify>
  <done>DicticusKeyboard target is correctly configured and project file generated.</done>
</task>

<task type="auto">
  <name>Implement URL Scheme Handler in DicticusApp</name>
  <files>iOS/Dicticus/DicticusApp.swift</files>
  <action>
    Add `.onOpenURL` to the `WindowGroup` in `DicticusApp.swift` to handle `dicticus://dictate?source=keyboard`.
    When triggered:
    1. Set `kbSource` flag in `UserDefaults(suiteName: "group.com.dicticus")` to `true`.
    2. Set `pendingDictation` to `true` (reusing existing logic).
    3. Post `.startDictation` notification if already active.
  </action>
  <verify>
    <automated>grep -q "onOpenURL" iOS/Dicticus/DicticusApp.swift</automated>
  </verify>
  <done>Main app responds to the keyboard bounce URL scheme.</done>
</task>
