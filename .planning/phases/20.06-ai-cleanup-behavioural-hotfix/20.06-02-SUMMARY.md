## What changed

### SwissNumberFormatter.foldCurrencyUnits (Shared/Utilities/SwissNumberFormatter.swift)

The existing fold patterns (which require BOTH primary unit + sub-unit) were already idempotent for spoken-out forms. The UAT regression was introduced by upstream paths emitting duplicate currency tokens (`110.57 â‚¬ Euro`, `Euro Euro`) that the formatter then re-saw on the next pipeline pass. Fix shape:

- Body wrapped with `// Phase 20.06 F-20-UAT-02: idempotency guard` comment block.
- After the existing 4-pattern fold, a post-fold sweep over `dedupePatterns` collapses adjacent duplicates:
  - Word-word: `\b(Euro|Franken|Dollar|Pfund|CHF|EUR|USD|GBP)\s+\1\b` (case-insensitive backreference)
  - Glyph-then-word same family: `â‚¬\s+Euro`, `\$\s+Dollar`, `ÂŁ\s+Pfund`
  - Word-then-glyph same family: `Euro\s+â‚¬`, `Dollar\s+\$`, `Pfund\s+ÂŁ`
- Each duplicate run is replaced with the LEADING token (preserves whichever form the user/LLM committed to first).
- All patterns are bounded â€” no nesting, no unbounded quantifiers â€” so backtracking is O(n).

### CurrencyAntiFlip (Shared/Utilities/CurrencyAntiFlip.swift)

New public method:

```swift
public static func speakerExplicitCurrencies(in text: String) -> Set<Family>
```

- Pattern: `(?<![\p{L}\p{N}])(Franken|Rappen|Euro|Dollar|Pfund|Pounds?)(?![\p{L}\p{N}])` (case-insensitive)
- Returns ONLY word-form mentions: glyphs (â‚¬/$/ÂŁ) and 3-letter codes (CHF/EUR/USD/GBP) are excluded â€” they do not count as "what the speaker said."
- Maps `franken|rappen â†’ .chf`, `euro â†’ .eur`, `dollar â†’ .usd`, `pfund|pound|pounds â†’ .gbp`.

`canonicalLabel(for:mirroring:preferWordForm:)` extended:
- New `preferWordForm: Bool = false` defaulted parameter â€” preserves existing call-sites.
- When `preferWordForm == true`, returns the WORD form (Franken/Euro/Dollar/Pfund) regardless of `mirror` shape.

`revertCurrencyFlip(input:output:)` extended:
- After family-set match check, splits into two paths:
  - `inputFamilies == outputFamilies && counts equal` â†’ return output unchanged (no-op preserved).
  - Counts mismatch â†’ bail (existing behavior).
  - Otherwise â†’ compute `speakerWords = speakerExplicitCurrencies(in: input)` and run positional revert with `preferWordForm: speakerWords.contains(targetFamily)`.

This implements: when speaker said "Franken" and LLM emitted "Euro" or "EUR" at the same positional index, restore the word "Franken" (not the code "CHF").
