<objective>
Fix F-20-UAT-01 (CRITICAL): the HELVETISMS prompt block in `Shared/Models/CleanupPrompt.swift`
currently tells Gemma 4 E2B to "Prefer these Swiss German words", which the model interprets
as a license to translate High German → Swiss German dialect (auf→uf, ausgeflogen→usgfloge,
gekostet→choschtet, etc.). UAT 2026-04-27 logged this regression on both platforms in
Configuration B + C.

Rework the block to be PRESERVATION-FIRST. Allow exactly two normalizations (ß→ss and
decimal-comma→period), preserve the existing positive Helvetism list (so the LLM still
has the vocabulary anchor when the speaker uses it), and add a NEGATIVE list enumerating
the common HG→CH-G traps the LLM must NOT apply.

Purpose: stop the LLM rewriting High German into Swiss German dialect when the Swiss
toggle is ON. Likely also mitigates F-20-UAT-05 (number drift "110.57"→"100.57") because
once Gemma stops rewriting words, it should stop touching adjacent digits too.

Output: 1 modified Shared/ file + 1-2 modified test files. Cross-platform: ships on
macOS AND iOS (`Shared/` is built into both targets per Phase 20 convention).
</objective>
