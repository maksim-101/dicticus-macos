<objective>
Fix two iOS history-row UX regressions logged in UAT 2026-04-27:

1. **F-20-UAT-03 (MEDIUM)** — Long-pressing a row pops a system path/link preview because SwiftUI `Text` auto-detects URL-shaped content (or falls through to the system long-press default when content is plain). Add a `.contextMenu` to `HistoryRow` so the long-press surfaces a Copy action we control instead of the system preview.

2. **F-20-UAT-04 (MEDIUM)** — `HistoryDetailView` shipped in Phase 20.05 but users don't discover the tap because there's no visible affordance. Add a trailing chevron to `HistoryRow` for parity with the macOS chevron added in 20.05.

Both fixes touch the same file (`iOS/Dicticus/History/HistoryView.swift`) and are tiny additive UI changes — combined into a single plan to avoid file-overlap with a parallel plan.

Output: 1 modified iOS file. No Shared/ changes (these are platform-specific UX affordances). No test infrastructure prerequisites — UAT verification is in Plan 20.06-04.
</objective>

| Boundary | Description |
|----------|-------------|
| user→UIPasteboard | The .contextMenu Copy button writes user-dictated text to the system pasteboard via `UIPasteboard.general.string`. This is the same boundary the existing inline Copy button already crosses; no new attack surface. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20.06.03-01 | Information Disclosure | UIPasteboard.general | accept | The user explicitly invokes Copy; the pasteboard is the user's own; no third-party access. Same risk profile as the existing Copy button. |
| T-20.06.03-02 | Tampering | NavigationLink tap stealing by chevron | mitigate | Chevron is a plain `Image`, not a Button — no `.onTapGesture`, no gesture modifier; cannot intercept NavigationLink. Verified by grep gate. |
| T-20.06.03-03 | Spoofing | .contextMenu Copy vs system Text auto-link | mitigate | The .contextMenu attached to the row root takes precedence over Text's URL auto-detection long-press on iOS 17+. This is the explicit fix for F-20-UAT-03. |
</threat_model>
