**Run date:** 2026-04-28
**Build:** macOS Debug, Cleanup Spike harness (Phase 20.08 Plan 03)

**Model:** Gemma 4 E2B IT Q4_K_M GGUF (production model used by CleanupService)

## Strategy pivot (mid-checkpoint, 2026-04-28)

The spike was originally designed around 4 variants (a/b/c/d) — each
preserving Swiss vocabulary while *adapting orthography*. First spike
run showed all four variants leaked Swiss dialect particles
(*Siite, wahrschiinli, alli, mini, uf, usgfloge, choschtet, gsi*) on the
2026-04-27 UAT fixture and on the everyday-German fixtures. Variant (d)
additionally collapsed under few-shot copy bias (rows 4 and 5 returned
text copied from the first few-shot example).

User reviewed the grid and pivoted the strategy:

> "what if we drop the Helvetisms or Swiss Dictionary and just see to it
> that the actual grammar, the orthography is in the Swiss German style.
> So no double S and the likes."

New cleanup contract:

- **Output Standard High German exactly as dictated.**
- **Apply Swiss orthography only** (`ss` instead of `ß`; thousands
  separator `1'250` not `1.250`).
- **No vocabulary substitution. No paraphrase. No dialect preservation.**

This re-aligns the prompt with the Plan 02 helvetism-delta gate, which
*punishes* dialect leakage — there is no longer any reason for the
prompt itself to ask for dialect words.

The spike harness was rewritten from a 5-column 4-variant grid to a
3-column 2-variant grid, with two new variants matching the new contract:

- **Variant (e):** single-rule "Standard High German + Swiss orthography
  only", no examples.
- **Variant (f):** variant (e) + non-biasing few-shot (one example per
  rule, diverse content so the model cannot copy any specific phrase).

Cosmetic post-processing was added (`cleanSpikeOutput`) to strip
`>` blockquote prefixes and to truncate at chat-template markers,
including the `</start_of_turn>` closing-tag variant.

## Fixtures

5. HG short subjunctive (known Swiss-form trigger):
   > "Ich wäre gerne früher zu Hause gewesen."

## Per-Variant Pass/Fail (final run, variants e + f)

Pass criterion: output preserves High German words AND grammar; no Swiss
dialect particles (uf/hie/het/alli/mini/wahrschiinli/usgfloge/gsi/…); ß
is replaced by ss; thousands separator follows Swiss orthography.

| Variant | F1 | F2 | F3 | F4 | F5 | Total |
|---------|----|----|----|----|----|-------|
| (e)     | ✅ | ✅ | ✅ | ✅ | ✅ | 5/5   |
| (f)     | ⚠️  | ✅ | ✅ | ❌ | ❌ | 2/5   |

Variant (f) failure mode confirmed the well-known few-shot copy bias
under low-temperature sampling on Gemma 4 E2B: rows 4 and 5 returned
"das war gross und teuer" — text copied verbatim from the first
few-shot example, ignoring the actual input. Row 1 lost punctuation /
capitalisation. Few-shot is unsafe at temp 0.1 on this model size.

Variant (e) preserved all five inputs as identity (no substitution
needed because the inputs were already Standard High German with no `ß`)
and produced no dialect leakage, no doubled output, and no markdown
artifacts after `cleanSpikeOutput` post-processing.

## Winner

**Variant: (e) — single-rule "Standard High German + Swiss orthography only".**

**Rationale:** Variant (e) is the only variant that satisfies the new
contract on every fixture. Few-shot exemplars (variant f) introduce
copy bias on Gemma 4 E2B at temp 0.1, contaminating outputs that look
nothing like the example inputs. Defense-in-depth: Plan 02's
helvetism-delta gate is the structural backstop if the prompt ever
slips at runtime.

## Verbatim Prompt Block to Ship in Plan 04

The block below is the *body* of the user-turn that goes between
`<start_of_turn>user\n` and `<end_of_turn>\n<start_of_turn>model`.
Plan 04 must drop this verbatim into `Shared/Models/CleanupPrompt.swift`,

## Notes / Open Issues

- **Stray `</start_of_turn>` closing-tag token** surfaced in fixture 1
  on the variant-(e) cell during the live spike run. Production
  `stripPreamble` handles this via regex; the spike post-processor was
  patched (commit `addac7d`) to strip closing-tag variants too. Issue is
  cosmetic and confined to the spike harness.

- **Currency spell-out conversion** ("vier Franken fünfzig" stays
  verbatim in fixture 4 under variant (e)) is **out of scope for Phase
  20.08**. The variant-(e) contract explicitly forbids paraphrase, so
  preserving the spelled-out form is correct behaviour for this prompt.
  Numeric conversion is a deterministic-ITN concern (sibling to Phase
  20.06's Swiss separator fix) — captured as a backlog item at
  `.planning/backlog/currency-spell-out-conversion.md`.

- **Few-shot at temp 0.1 on Gemma 4 E2B is unsafe.** Variant (f) failure
  is reproducible — first-example copy bias overrides the actual input
  on at least 2/5 fixtures. Future prompt iterations should avoid
  exemplars at this temperature on this model.

- **SwissDialectForms.tokens (Plan 01)** was sized for the original
  helvetism-preservation contract. Under the new contract the token
  list is no longer used by the *prompt*, but it remains the source of
  truth for the helvetism-delta *gate* (Plan 02). Token list stays as
  shipped in Wave 1 — no change.

- **Plan 02's helvetism-delta gate** is the structural backstop. Even
  if the LLM occasionally produces a Swiss particle under live sampling,
  the gate's similarity check rejects outputs that drift further from
  the input than the configured Levenshtein band. This is the
  defense-in-depth that lets us ship a single-rule prompt with
  confidence.

## Wave B Update (2026-04-30): Variant (g15) is the final winner

The variant (e) decision above is preserved verbatim as historical
record (per `20.08-VARIANT-G-RATIONALE.md` §8.1). Post-spike UAT on
multi-clause real input exposed three failure modes in variant (e) that
the 5-fixture spike never surfaced (lowercase nouns, "und Und"
clause-seam duplication, "Würdigten" meaning-inference scar that
identity-preservation forbade repairing). See VARIANT-G-RATIONALE §1.5
for the empirical record.

The cleanup contract was reframed (VARIANT-G-RATIONALE §1) and Wave B
ran a new spike under the (g) family. After several iterations
(g → g12 in `/tmp/spike-harness` → g14 disqualified for header bloat →
**g15**), the final winner is variant (g15). It uses a few-shot
ORIGINAL/KORRIGIERT frame (4 examples, German), no negative-particle
list, and a six-token umlaut-preservation clause that fixes the F3
ASCII-fold drift seen in g12.

**Multi-seed verification at the production sampler (temp 0.1,
top-k 40, top-p 0.9, seed 0xDEADBEEF):**

- g12 → 8/8 muessen ASCII-fold drift on F3 (deterministic failure)
- g14 → disqualified (verbose 4-pair umlaut anchor caused F2
  catastrophic prompt-leak and Deadline→Frist on F3)
- **g15 → 0/8 muessen drift; 7/7 fixtures pass full sweep**

### Verbatim Prompt Block to Ship in Plan 04 (variant g15)

The block below is the full user-turn body, written in German (the
g-family pivot from English instructions to native-language directive
was confirmed empirically more stable on Gemma 4 E2B). Plan 04 must
drop this verbatim into `Shared/Models/CleanupPrompt.swift` per the
two-layer German conditional from VARIANT-G-RATIONALE §4 D3:
- `language == "de"` → variant (g15) INSTRUCTION + 4-shot frame +
  RULE 1 (Standard High German) appended
- `swissEnabled && language == "de"` → RULE 2 (Swiss orthography)
  embedded as the `ss statt ß, Umlaute ä/ö/ü bleiben` clause inside
  the INSTRUCTION line (already present in g15)

```text
<start_of_turn>user
Bereinige die folgende deutsche Sprachaufnahme. Schreibe Standard-Hochdeutsch mit Schweizer Rechtschreibung (ss statt ß, Umlaute ä/ö/ü bleiben). Verwende KEINEN Schweizerdeutsch-Dialekt — schreibe "Woche" nicht "Wuche", "Zürich" nicht "Züri", "ich gehe" nicht "i gang". Etablierte englische Fachbegriffe bleiben Englisch (Deadline, Meeting, Workaround, E-Mail, Team, Product Owner, Release). Untypische englische Adjektive oder Verben in deutschen Sätzen ins Deutsche übertragen — "realistic" → "realistisch", "awesome" → "toll", "appreciate" → "schätzen". Gib genau eine bereinigte Version aus, sonst nichts.

ORIGINAL: ich habe heute mit dem product owner gesprochen über die deadline und er meinte das ist nicht realistic.
KORRIGIERT: Ich habe heute mit dem Product Owner über die Deadline gesprochen, und er meinte, das ist nicht realistisch.

ORIGINAL: ich gestern gehen markt und kaufen viele apfel weil ich brauchen für kuchen.
KORRIGIERT: Ich bin gestern auf den Markt gegangen und habe viele Äpfel gekauft, weil ich sie für den Kuchen brauche.

ORIGINAL: das hotel hat ungefähr 1250 franken 20 gekostet und das war zu viel.
KORRIGIERT: Das Hotel hat ungefähr 1250 Franken 20 gekostet, und das war zu viel.

ORIGINAL: letzte woche war ich in zürich auf einer grossen konferenz.
KORRIGIERT: Letzte Woche war ich in Zürich auf einer grossen Konferenz.

ORIGINAL: <user input here>
KORRIGIERT:<end_of_turn>
<start_of_turn>model
```

The reference Swift builder is `SpikeFixtures.buildVariantG15` in
`macOS/Dicticus/Views/CleanupSpikeView.swift`.

**Apostrophe-strike (commit `435004b`):** g15 contains NO thousands
separator clause and NO apostrophe in the third example. Deterministic
Swift (`SwissNumberFormatter`) emits ungrouped integers post-LLM, so
any apostrophe the LLM might insert would be redundant noise.

### Why g15 Beats g (the in-app variant)

`SpikeFixtures.buildVariantG` in CleanupSpikeView (the original
instruction-only variant g) uses a longer English instruction with a
RULE 3 cardinals-to-digits clause. It shipped to the in-app harness
during Wave-B exploration but was superseded by g15 once the
`/tmp/spike-harness` round-trip showed:
- Few-shot ORIGINAL/KORRIGIERT pairs stabilise Gemma 4 E2B at temp 0.1
  when each example demonstrates a *distinct* rule (anglicism
  preservation, broken-grammar reconstruction, currency cents
  preservation, dialect-anti-drift). This contradicts the variant (f)
  conclusion above — diversity in *rule coverage* beats diversity in
  *content* for this model size.
- A native-language (German) instruction body avoids the
  language-switch token cost the English RULE-list version paid.
- The RULE 3 cardinals-to-digits clause was moved out of the prompt
  into deterministic Swift (`SwissNumberFormatter` Bridge 1.5,
  commit `17005f3`).

### Defense-in-Depth Stack (Final)

1. Deterministic Swift pre-pass (`RulesCleanupService`,
   `ITNUtility.applyITN`) — disfluencies, fillers, cardinals ≥10.
2. Variant (g15) LLM cleanup — grammar, capitalisation, punctuation,
   gap-filling, contextual mishear repair.
3. `SwissNumberFormatter` post-pass — Swiss orthography for digits,
   currency cents fold (Bridge 1.5 + Bridge 2 with widened regex).
4. `CleanupService.gateLLMDialect` (Plan 02) — structural backstop:
   demotes to `rulesCleanedText` if the LLM injects a Swiss-dialect
   token absent from the raw ASR.
