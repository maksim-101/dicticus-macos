# Phase 20.08 Spike Results

**Run date:** 2026-04-28
**Build:** macOS Debug, Cleanup Spike harness (Phase 20.08 Plan 03)
**Sampler seed:** 0xDEADBEEF (pinned via `CleanupService.setSamplerSeed`)
**Model:** Gemma 4 E2B IT Q4_K_M GGUF (production model used by CleanupService)

## Strategy pivot (mid-checkpoint, 2026-04-28)

The spike was originally designed around 4 variants (a/b/c/d) — each
preserving Swiss vocabulary while *adapting orthography*. First spike
run showed all four variants leaked Swiss dialect particles
(*Siite, wahrschiinli, alli, mini, uf, usgfloge, choschtet, gsi*) on the
2026-04-27 UAT fixture and on the everyday-German fixtures. Variant (d)
additionally collapsed under few-shot copy bias (rows 4 and 5 returned
text copied from the first few-shot example).

User reviewed the grid and pivoted the strategy:

> "what if we drop the Helvetisms or Swiss Dictionary and just see to it
> that the actual grammar, the orthography is in the Swiss German style.
> So no double S and the likes."

New cleanup contract:

- **Output Standard High German exactly as dictated.**
- **Apply Swiss orthography only** (`ss` instead of `ß`; thousands
  separator `1'250` not `1.250`).
- **No vocabulary substitution. No paraphrase. No dialect preservation.**

This re-aligns the prompt with the Plan 02 helvetism-delta gate, which
*punishes* dialect leakage — there is no longer any reason for the
prompt itself to ask for dialect words.

The spike harness was rewritten from a 5-column 4-variant grid to a
3-column 2-variant grid, with two new variants matching the new contract:

- **Variant (e):** single-rule "Standard High German + Swiss orthography
  only", no examples.
- **Variant (f):** variant (e) + non-biasing few-shot (one example per
  rule, diverse content so the model cannot copy any specific phrase).

Cosmetic post-processing was added (`cleanSpikeOutput`) to strip
`>` blockquote prefixes and to truncate at chat-template markers,
including the `</start_of_turn>` closing-tag variant.

## Fixtures

1. UAT-anchor (2026-04-27 failing utterance):
   > "Auf der anderen Seite war ich wahrscheinlich, alle meine, würde ich denn, hat, hier ausfiltern."
2. Short HG everyday:
   > "Heute war ich auf dem Markt einkaufen."
3. Long HG multi-clause:
   > "Ich bin am Freitag ausgeflogen und habe etwas gekauft, das natürlich teurer war als gedacht."
4. HG with currency (intersection with Phase 20.06 anti-flip):
   > "Das hat mich dann ungefähr vier Franken fünfzig gekostet."
5. HG short subjunctive (known Swiss-form trigger):
   > "Ich wäre gerne früher zu Hause gewesen."

## Per-Variant Pass/Fail (final run, variants e + f)

Pass criterion: output preserves High German words AND grammar; no Swiss
dialect particles (uf/hie/het/alli/mini/wahrschiinli/usgfloge/gsi/…); ß
is replaced by ss; thousands separator follows Swiss orthography.

| Variant | F1 | F2 | F3 | F4 | F5 | Total |
|---------|----|----|----|----|----|-------|
| (e)     | ✅ | ✅ | ✅ | ✅ | ✅ | 5/5   |
| (f)     | ⚠️  | ✅ | ✅ | ❌ | ❌ | 2/5   |

Variant (f) failure mode confirmed the well-known few-shot copy bias
under low-temperature sampling on Gemma 4 E2B: rows 4 and 5 returned
"das war gross und teuer" — text copied verbatim from the first
few-shot example, ignoring the actual input. Row 1 lost punctuation /
capitalisation. Few-shot is unsafe at temp 0.1 on this model size.

Variant (e) preserved all five inputs as identity (no substitution
needed because the inputs were already Standard High German with no `ß`)
and produced no dialect leakage, no doubled output, and no markdown
artifacts after `cleanSpikeOutput` post-processing.

## Winner

**Variant: (e) — single-rule "Standard High German + Swiss orthography only".**

**Rationale:** Variant (e) is the only variant that satisfies the new
contract on every fixture. Few-shot exemplars (variant f) introduce
copy bias on Gemma 4 E2B at temp 0.1, contaminating outputs that look
nothing like the example inputs. Defense-in-depth: Plan 02's
helvetism-delta gate is the structural backstop if the prompt ever
slips at runtime.

## Verbatim Prompt Block to Ship in Plan 04

The block below is the *body* of the user-turn that goes between
`<start_of_turn>user\n` and `<end_of_turn>\n<start_of_turn>model`.
Plan 04 must drop this verbatim into `Shared/Models/CleanupPrompt.swift`,
replacing the existing HELVETISMS block at lines ~76-92. Surrounding
chat-template scaffolding (`<start_of_turn>` / `<end_of_turn>` markers,
`LANGUAGE: de` / `LANGUAGE: en` switch, the `INPUT:` and `OUTPUT:`
labels with the actual user input interpolation) stays as it is in
production.

```text
INSTRUCTION: Edit the following German dictation for grammar, punctuation, and capitalization. Do not paraphrase. Do not substitute any word with a different word.
LANGUAGE: de
RULE 1: Output Standard High German words and grammar exactly as dictated.
RULE 2: Apply Swiss orthography only — write ss instead of ß; write thousands as 1'250 not 1.250.
```

For reference, the full variant-(e) prompt as built by
`SpikeFixtures.buildVariantE` in `CleanupSpikeView.swift`:

```text
<start_of_turn>user
INSTRUCTION: Edit the following German dictation for grammar, punctuation, and capitalization. Do not paraphrase. Do not substitute any word with a different word.
LANGUAGE: de
RULE 1: Output Standard High German words and grammar exactly as dictated.
RULE 2: Apply Swiss orthography only — write ss instead of ß; write thousands as 1'250 not 1.250.
INPUT: <user input here>
OUTPUT:<end_of_turn>
<start_of_turn>model
```

Plan 04 must respect the existing language-switch logic
(`LANGUAGE: de` vs `LANGUAGE: en`) and continue interpolating the
actual `INPUT:` value and the `OUTPUT:` placeholder per the production
prompt assembly. Only the *body* — the INSTRUCTION line and the two
RULE lines — is changing relative to the pre-20.08 HELVETISMS block.

## Notes / Open Issues

- **Stray `</start_of_turn>` closing-tag token** surfaced in fixture 1
  on the variant-(e) cell during the live spike run. Production
  `stripPreamble` handles this via regex; the spike post-processor was
  patched (commit `addac7d`) to strip closing-tag variants too. Issue is
  cosmetic and confined to the spike harness.

- **Currency spell-out conversion** ("vier Franken fünfzig" stays
  verbatim in fixture 4 under variant (e)) is **out of scope for Phase
  20.08**. The variant-(e) contract explicitly forbids paraphrase, so
  preserving the spelled-out form is correct behaviour for this prompt.
  Numeric conversion is a deterministic-ITN concern (sibling to Phase
  20.06's Swiss separator fix) — captured as a backlog item at
  `.planning/backlog/currency-spell-out-conversion.md`.

- **Few-shot at temp 0.1 on Gemma 4 E2B is unsafe.** Variant (f) failure
  is reproducible — first-example copy bias overrides the actual input
  on at least 2/5 fixtures. Future prompt iterations should avoid
  exemplars at this temperature on this model.

- **SwissDialectForms.tokens (Plan 01)** was sized for the original
  helvetism-preservation contract. Under the new contract the token
  list is no longer used by the *prompt*, but it remains the source of
  truth for the helvetism-delta *gate* (Plan 02). Token list stays as
  shipped in Wave 1 — no change.

- **Plan 02's helvetism-delta gate** is the structural backstop. Even
  if the LLM occasionally produces a Swiss particle under live sampling,
  the gate's similarity check rejects outputs that drift further from
  the input than the configured Levenshtein band. This is the
  defense-in-depth that lets us ship a single-rule prompt with
  confidence.
