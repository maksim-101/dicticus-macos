empirical loop than rebuilding the macOS app for every prompt change:

in production, against the same post-RulesCleanupService input. The
harness output is summarised cell-by-cell in `summarize.py` for
capitalisation, digit preservation, `√Я`-eszett regression, and Swiss
particle leakage. This bridged the test/prod gap that Plan 03's spike
harness papered over (R6 contract tests assert prompt structure but
never invoke the model).

### Known limitations carried forward (NOT blocking)

1. **Sentence-stitching across ASR boundaries** вАФ Gemma occasionally
   merges two source sentences via comma when adjacent clauses appear
   syntactically continuable. Below user's acceptance bar.
2. **Tense correction is non-deterministic** вАФ depends on Parakeet
   emitting `Gestern` at sentence start. When Parakeet merges the
   `gestern` into a prior clause, the iteration-2 trigger pattern is
   not present in the runtime ORIGINAL.
3. **R-G15-02 (iOS-only language-detection bleed into preamble)** вАФ
   unchanged from Plan 04 UAT, upstream of the prompt, requires a
   separate phase.

### Recurrence handling

If R-G15-01 reproduces in a future UAT, do NOT re-introduce a negative-
instruction directive ("don't truncate digits"). The iteration-1 80-cell
harness data established that approach as a ¬І3 priming trap with zero
measurable benefit. Preferred next moves:

1. Add a second decimal-currency exemplar with a different magnitude
   (e.g. 4-digit decimal `1234.56 Franken`) before the anchor 6th.
2. Replace the runtime-ORIGINAL preceding context one slot earlier with
   another currency-identity exemplar.

Both keep the recency-bias contract intact and the priming-trap shut.
