## Sampling Rate

- **After every task commit:** Run targeted unit test for the file just touched (e.g. `…SwissDialectFormsTests` after editing `SwissDialectForms.swift`).
- **After every plan wave:** Run full macOS suite. Run iOS suite at the wave boundary IF the FluidAudio drift is resolved (see Manual-Only).
- **Before `/gsd-verify-work`:** Full macOS suite green; iOS suite green OR explicit deferral note documented.
- **Max feedback latency:** ≤ 60s for targeted unit; ≤ 300s for full suite.

## Wave 0 Requirements

> Existing infrastructure covers all phase requirements. The XCTest framework, both `iOS/DicticusTests/` and `macOS/DicticusTests/` targets, and the `Shared/` cross-platform module are all already wired and exercised by Phase 20.06. Phase 20.08 introduces NEW test files (`SwissDialectFormsTests`, gate-function tests in `CleanupServiceTests`, integration test in `TextProcessingServiceTests`) but ships them inside the same plan as the implementation they verify (TDD-within-plan), so no infrastructure pre-step is required.
>
> Pre-execution sanity check (one-time, before /gsd-execute-phase):
> - Confirm `cd iOS && xcodegen generate && xcodebuild build -scheme Dicticus -destination 'platform=iOS Simulator,name=iPhone 17' -quiet` succeeds. If the deferred `IOSTranscriptionService.swift` FluidAudio drift from Phase 20.06 is unresolved, iOS test execution may be blocked — document the deferral and proceed macOS-only for unit tests; iOS UAT (R9) is still required at phase gate.
