# Phase 20.08: LLM Swiss-Ification Suppression - Context

**Gathered:** 2026-04-27
**Status:** Ready for planning

<domain>
## Phase Boundary

Stop Gemma 4 E2B from rewriting clean Standard High German into Swiss German dialect on the AI-cleanup ON + Swiss-toggle ON path. The 2026-04-27 UAT (post-20.06 ship) showed the speaker dictating clean High German and the LLM emitting `uf de andere Siite, wahrschiinli, alli mini, wuer ich denn, het, hie usfiltere`. Phase 20.06's preservation-first prompt (CleanupPrompt.swift:83) is being ignored by the model.

**Clarified scope:** AI-cleanup output must always be Standard High German words and grammar (Swiss orthography conventions for ß and numerics remain when the toggle is on — those are the toggle's only legitimate effect). High-German input → Swiss output is the only failure mode being addressed in this phase. Swiss-as-input handling is NOT in scope: the user does not dictate Swiss German into the mic, so the "speaker actually said Swiss, ASR flattened it, LLM rescues dialect" edge case is hypothetical and excluded.

**In scope:**
- Empirical mini-spike to identify which prompt levers actually drive the Swiss-ification (not assumed from roadmap hypothesis)
- Prompt restructure based on spike winner
- New helvetism-delta gate (pre-Levenshtein) that demotes LLM output when it injects Swiss dialect tokens absent from raw ASR
- New `SwissDialectForms.swift` token list distinct from existing `SwissHelvetisms.words`
- Fixture corpus + unit tests on the gate; manual UAT replay for the prompt
- Cross-platform parity (macOS + iOS — `Shared/Models/CleanupPrompt.swift` + `Shared/Services/CleanupService.swift`)

**Out of scope:**
- Swiss-input → High-German-output normalization (not a real scenario for this user)
- Replacing or weakening the existing Levenshtein gate (`gateLLMOutput`, threshold 0.30)
- New History UI / demotion indicator (gate stays invisible to match Levenshtein-gate behaviour)
- Phase 20.07 work (rules-only ASR mishearing recovery — separate phase)

</domain>

<decisions>
## Implementation Decisions

### Empirical-First Approach
- **D-01:** The roadmap hypothesis ("`SwissHelvetisms.words` reference list is the priming source") is unproven. Phase 20.08 starts with an empirical mini-spike before locking the prompt. The spike's winner drives the prompt restructure; we do not ship a prompt change on hypothesis alone.
- **D-02:** Spike harness = on-device, manual judge. Run on macOS Debug build via a temporary debug menu / test scaffold. Outputs side-by-side; user visually picks the variant that preserves High German best.
- **D-03:** Spike corpus = curated 5 inputs × N prompt variants, seed pinned where the runtime allows. Five inputs: the 2026-04-27 UAT failure utterance + 4 variations covering short-utterance / long-utterance / pure-High-German-everyday / High-German-with-currency. Total ~20–30 inferences, target <2h end-to-end.
- **D-04:** Prompt variants under test: (a) drop `SwissHelvetisms.words` reference list entirely, keep negative trap list; (b) drop trap list, keep reference list; (c) drop both, minimal preservation-first instruction only; (d) variants (a) + few-shot positive examples (input `auf der anderen Seite` → output `auf der anderen Seite`). Final variant set may expand based on spike findings.
- **D-05:** Prompt instruction reframed away from "preserve speaker's dialect register exactly" (which leaks bidirectional intent) toward an unambiguous one-way directive: output must always be Standard High German words and grammar. Swiss orthography (`ss`/numerics) stays in the STYLE block — separate from word/grammar choice.

### Helvetism-Delta Gate
- **D-06:** Detection mechanism = curated dialect-form token list. Detect actual offending Swiss FORMS that the LLM injects (e.g. `uf, hie, het, alli, mini, wahrschiinli, isch, nid, gsi, gseh, gha, ...`), NOT substantive Helvetisms (Velo, Trottoir, parkieren — those are already handled by `SwissHelvetisms.words` for orthography purposes). Heuristic regex/morphology approaches rejected (false-positive-prone).
- **D-07:** Trigger threshold = strict delta ≥ 1. If LLM output contains ≥1 dialect form NOT present in raw ASR → demote to `rulesCleaned`. Aligned with the threat model: the user does not dictate Swiss, so the "speaker actually used dialect" edge case does not occur. Any unsolicited Swiss-ification is rejected.
- **D-08:** Gate stacking = pre-gate position. Helvetism-delta runs FIRST on raw LLM output, BEFORE the Levenshtein gate's normalization step. If helvetism-delta demotes, Levenshtein gate is skipped (already returning `rulesCleaned`). If helvetism-delta passes, Levenshtein gate runs unchanged as the structural fail-safe for hallucinations / over-rewrites. Both gates demote to the same target (`rulesCleaned`).
- **D-09:** Token list location = new file `Shared/Models/SwissDialectForms.swift`, public constant `SwissDialectForms.tokens: [String]`. Curated from the 2026-04-27 UAT failure transcript + common Swiss particles. Distinct from `SwissHelvetisms.words` (substantive helvetisms, retained for orthography prompt block only). Single call site: the new gate function in `CleanupService.swift`.
- **D-10:** Demotion is invisible to the user. Match existing Levenshtein-gate behaviour: silent fall-through to `rulesCleaned`. No History badge, no notification. Diagnostic logging is acceptable in Debug builds (`os_log` with offending tokens) for tuning but is NOT a user-facing feature.

### Fixture Corpus + Tests
- **D-11:** Fixture anchor = the actual 2026-04-27 UAT failing utterance (`uf de andere Siite, wahrschiinli, alli mini, wuer ich denn, het, hie usfiltere` and its source High-German input) + 4–5 synthesized High-German inputs likely to trip Gemma into Swiss mode. Each fixture pairs raw input with expected demote/keep verdict.
- **D-12:** Test harness split:
  - Helvetism-delta gate gets pure-Swift unit tests in the existing macOS + iOS test targets (`iOS/DicticusTests/`, `macOS/DicticusTests/` if present). Tests construct synthetic raw/llm pairs against the new gate function. No GGUF loaded in unit tests.
  - Prompt-variant scoring stays in the manual on-device spike harness from D-02. Two harnesses, two purposes — kept separate to avoid bringing the LLM into CI.
- **D-13:** Cross-platform parity (per memory `feedback_cleanup_cross_platform_parity.md`): the gate function and `SwissDialectForms.swift` live in `Shared/`; tests run on both targets where the test target exists.

### Cross-Cutting
- **D-14:** Swiss toggle semantics clarified in this phase: the toggle controls **orthography** (ß→ss, `1'250` numeric style, currency formatting) and nothing more. It does NOT and MUST NOT cause word/grammar Swiss-ification. The HELVETISMS prompt block carries this clarification too — it is no longer about "register preservation."
- **D-15:** No replacement of the Levenshtein gate. `gateLLMOutput` (threshold 0.30) and `levenshteinGateThreshold` constant remain as-is — they catch hallucinations and over-rewrites, which the helvetism-delta gate does not.

### Claude's Discretion
- Concrete prompt wording for the spike variants (a)–(d) — Claude drafts; user reviews spike outputs.
- Initial token list contents for `SwissDialectForms.tokens` — Claude curates from the UAT transcript + linguistically obvious Swiss particles; user can prune or extend during planning if needed.
- Naming of the new gate function (e.g. `gateLLMOutputDialect` vs. `helvetismDeltaGate`) — Claude picks consistent with existing `gateLLMOutput`.
- Whether the spike's debug menu/scaffold ships in Debug-only builds or gets removed before merging — Claude decides during execution based on cleanliness.

</decisions>

<specifics>
## Specific Ideas

- **Spike anchor input** = the literal 2026-04-27 UAT transcript (`uf de andere Siite, wahrschiinli, alli mini, wuer ich denn, het, hie usfiltere`) paired with its known clean-High-German source. This is fixture #1 and the spike's primary regression check.
- **Few-shot example pattern** (if D-04 variant (d) wins): positive only — input `auf der anderen Seite` → output `auf der anderen Seite`. No negative-with-strikethrough examples (those tend to prime the wrong substitution).
- **STYLE-only Swiss block** (after the prompt restructure): the `swissEnabled && language == "de"` branch should keep ONLY the orthography line (`Use Swiss German orthography (never use ß, always ss). Use Swiss thousands separator style (e.g. 1'250, not 1.250).`) plus a one-liner negative directive. The trap-pair list and the `SwissHelvetisms.words` reference list are evaluated for removal by the spike.
- **One-way directive replacing "preserve register"**: explicit phrasing along the lines of "Output Standard High German words and grammar. Apply Swiss orthography conventions only" — avoids the bidirectional read of "preserve."

</specifics>

- **Adding heuristic morphology to the helvetism-delta detection** (e.g. `-li$` suffix detection, word-initial `ch` for `g`/`k`). Rejected for 20.08 (false-positive risk). Future iteration if the curated token list proves insufficient.
