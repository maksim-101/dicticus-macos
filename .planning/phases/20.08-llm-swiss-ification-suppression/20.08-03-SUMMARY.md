# Phase 20.08 Plan 03: Prompt Spike Harness Summary

DEBUG-only A/B-style harness for empirically picking the Gemma 4 E2B
prompt that best preserves Standard High German under Swiss orthography.
Built as a 5-fixture grid against the production CleanupService inference
path with a pinned sampler seed (`0xDEADBEEF`). Mid-checkpoint the user
pivoted the strategy from "preserve helvetism vocabulary" to "Standard
High German + Swiss orthography only, no paraphrase". The harness was
re-shaped from 4 variants to 2 (e + f), and **variant (e) won 5/5
fixtures**. The verbatim prompt block is recorded in
`20.08-SPIKE-RESULTS.md` ready for Plan 04 to drop into
`CleanupPrompt.swift`.

## Strategy Pivot

The plan's CONTEXT.md framed prompt-engineering as "preserve Swiss
vocabulary by listing helvetisms in the prompt as positive guidance".
First spike disproved this approach — Gemma 4 E2B treated the
helvetism reference list as a *hint to dialect-ify*, producing
*Siite, wahrschiinli, alli, mini, uf, usgfloge, choschtet, gsi* across
all 4 variants, including on inputs that contained no Swiss vocabulary
at all.

User reviewed the grid and re-articulated the cleanup contract:

> "what if we drop the Helvetisms or Swiss Dictionary and just see to
> it that the actual grammar, the orthography is in the Swiss German
> style. So no double S and the likes."

This re-aligns the prompt with the *Plan 02 helvetism-delta gate*: the
gate punishes dialect leakage, so the prompt should not request dialect
preservation in the first place. Plan 02 becomes the structural
backstop; the prompt becomes the smallest possible single-rule
contract.

## Variant (e) — the Winner

```text
INSTRUCTION: Edit the following German dictation for grammar, punctuation, and capitalization. Do not paraphrase. Do not substitute any word with a different word.
LANGUAGE: de
RULE 1: Output Standard High German words and grammar exactly as dictated.
RULE 2: Apply Swiss orthography only — write ss instead of ß; write thousands as 1'250 not 1.250.
```

**Why it won:**

- 5/5 fixtures preserved as identity (correct behaviour for inputs that
  were already Standard High German with no `ß` and no thousands
  separator).
- Zero Swiss dialect leakage.
- No few-shot bias (no exemplars).
- Smallest surface area for future maintenance — single rule, no token
  list to drift, no exemplars to mis-tune.

Plan 04 ships this body verbatim into
`Shared/Models/CleanupPrompt.swift`, replacing the existing HELVETISMS
block (~lines 76-92). Surrounding scaffolding
(`<start_of_turn>` markers, language switch, INPUT/OUTPUT
interpolation) stays as it is in production.

## Variant (f) Failure

Variant (f) added a 3-line non-biasing few-shot to variant (e). On
paper this should have helped: each example demonstrates exactly one
rule (ß→ss, identity preservation, thousands separator) with diverse
content so the model cannot copy any specific phrase. Empirically it
failed:

- Row 1: dropped capitalisation / punctuation (wrong rule applied).
- Rows 4 and 5: returned "das war gross und teuer" — verbatim copy of
  the first few-shot example, ignoring the actual input entirely.

Conclusion: few-shot exemplars are unsafe on Gemma 4 E2B at temp 0.1.
The model's first-example bias overwhelms input attention. Future
prompt iterations should avoid exemplars at this temperature on this
model size.

## LSUIElement / Menu-Bar Surfacing

Dicticus runs with `LSUIElement: true` (agent app, no Dock icon, no
top menu bar). The plan's `CommandGroup(after: .windowList)` keyboard
shortcut `⌘⇧⌥K` registered correctly but had no visible surface and
did not fire from a menu-bar-only app context. Workaround:
DEBUG-only popover button in `MenuBarView.swift`
(`Button("Cleanup Spike (Debug)…") { openWindow(id: "cleanup-spike") }`),
gated by `#if DEBUG`. User confirmed the spike window opens from the
popover.

## Cosmetic Post-Processing Bug

First (e/f) run surfaced a stray `</start_of_turn>` closing-tag token
on fixture 1 of variant (e). Production `stripPreamble` handles this
via regex (`</?(?:start_of_turn|end_of_turn)>`); the spike harness's
`cleanSpikeOutput` used substring matching on the bare opening tag
which skipped the closing-tag variant. Patched in `addac7d`: marker
list now includes `</start_of_turn>` and `</end_of_turn>` ahead of the
opening tags. Issue is confined to the spike harness — production
inference is unaffected.

## Deviations from Plan

### Plan 04 Inputs Changed Shape

Plan 04 was originally wired around the 4-variant matrix
("variants a/b/c/d"). The pivot shifts the input to a single
single-rule prompt body. Plan 04's task spec must consume the
"Verbatim Prompt Block to Ship in Plan 04" section of
`20.08-SPIKE-RESULTS.md` rather than reading any specific letter
variant. Plan 04 acceptance criteria already reference
"the chosen variant" rather than a specific letter, so the schema
holds — only the source of the verbatim block changed.

### Currency Spell-Out Surfaced in Fixture 4

Variant (e) preserves "vier Franken fünfzig" verbatim in fixture 4 —
correct behaviour for the new identity-preserving contract. The user
flagged this as something to address but it is **out of scope for
Phase 20.08**: numeric conversion is deterministic-ITN territory
(`Shared/Utilities/ITNUtility.swift`), sibling to Phase 20.06's Swiss
separator fix. Captured at `.planning/backlog/currency-spell-out-conversion.md`
as a candidate for a future ITN-polish phase.
